## 1. 데이터 분석과 컴퓨터

> 교재 대응: 교재 14쪽 예제 1.1 KOSIS 시계열 분산형 차트(척도 구분과 분석 워크플로우 도입부).

### 개요: 왜 배우나

데이터 분석은 숫자와 텍스트로 쌓인 자료에서 의미 있는 패턴과 결론을 끌어내는 일이다. 1장은 본격적인 도구(엑셀/R/Python) 실습에 앞서, 그 도구로 무엇을 다루는지를 정리하는 단계다. 데이터의 종류(척도)를 구분할 줄 알아야 어떤 통계량과 그래프를 쓸지 정할 수 있고, 모집단과 표본을 구분할 줄 알아야 "내가 가진 데이터로 어디까지 말할 수 있는가"를 판단할 수 있다. 즉 1장은 이후 모든 장의 공통 문법이다. 척도를 잘못 보면 평균을 내면 안 되는 데이터에 평균을 내는 식의 오류가 생긴다.

핵심 질문 세 가지로 요약된다.
- 이 데이터는 어떤 타입인가? (계산해도 되는 숫자인가, 이름표일 뿐인 숫자인가)
- 이 데이터는 전체인가, 일부인가? (모집단 vs 표본)
- 무엇을 거쳐 결론에 도달하는가? (분석 워크플로우)

### 핵심 개념·기능

- **데이터 분석(Data Analysis)**: 수집한 자료를 정리/요약/시각화/모형화하여 의사결정에 필요한 정보를 얻는 과정.
- **변수(variable)와 관측치(observation/case)**: 변수는 측정 항목(키, 성별 등), 관측치는 한 개체에 대한 측정값의 묶음(한 사람의 한 행).
- **데이터 타입 큰 분류**
    - 범주형(categorical/qualitative): 분류·이름을 나타냄. 빈도·비율로 요약.
    - 연속형(numerical/quantitative): 수량을 나타냄. 평균·표준편차 등으로 요약. (이산형/연속형으로 더 나뉨)
- **측정 척도(4가지 수준)**
    - 명목척도(nominal): 순서 없는 분류. 예) 성별, 혈액형, 지역코드. → 최빈값만 의미.
    - 순서척도(ordinal): 순서는 있으나 간격은 일정치 않음. 예) 만족도(상/중/하), 학점, 선호 순위. → 중앙값까지 의미.
    - 등간척도(interval): 간격이 일정하지만 절대 0이 없음. 예) 섭씨 온도, 연도. → 덧셈/뺄셈 의미, 비율(배수) 무의미.
    - 비율척도(ratio): 절대 0이 있어 배수 비교 가능. 예) 키, 몸무게, 소득, 나이. → 모든 사칙연산 의미.
- **모집단(population)과 표본(sample)**: 모집단은 관심 대상 전체, 표본은 거기서 뽑은 일부. 모집단의 특성치는 모수(parameter), 표본의 특성치는 통계량(statistic).
- **기술통계 vs 추론통계**: 기술통계는 가진 데이터를 요약/묘사, 추론통계는 표본으로 모집단을 추정/검정.
- **분석 워크플로우**: 문제 정의 → 데이터 수집 → 정제(전처리) → 탐색(EDA) → 분석/모형화 → 해석/시각화 → 보고.

암기 팁(척도 4단계): 명목 < 순서 < 등간 < 비율 순으로 "정보량"이 커진다. 위 단계의 연산은 아래 단계에서도 항상 가능하다(비율은 다 됨).

### 단계별 실습

#### 실습 A. KOSIS에서 실제 데이터를 받아 척도를 분류해 보기

1. 웹브라우저에서 KOSIS 국가통계포털(kosis.kr)에 접속한다.
2. 상단 검색창에 "시도별 인구"처럼 관심 주제를 입력하고 통계표를 하나 연다.
3. 통계표 화면에서 항목/분류/시점을 고른 뒤 다운로드 버튼을 눌러 EXCEL 또는 CSV로 저장한다. (셀 수가 매우 많으면 통계목록의 "파일다운로드"를 쓴다.)
4. 받은 파일을 엑셀로 열고, 첫 행의 열(변수)들을 종이에 적는다.
5. 각 변수 옆에 척도를 적는다. 예) "시도명=명목", "연도=등간", "인구수=비율", "만족도등급(있다면)=순서".
6. 각 변수를 요약할 때 평균이 의미 있는지(연속형/비율·등간) 빈도가 의미 있는지(범주형/명목·순서)를 함께 메모한다.

#### 실습 B. 엑셀로 기술통계 손으로 내보기

1. 실습 A에서 받은 "인구수" 같은 연속형 열을 하나 고른다.
2. 빈 셀에 `=AVERAGE(범위)`(평균), `=MEDIAN(범위)`(중앙값), `=STDEV.S(범위)`(표준편차), `=COUNT(범위)`(개수)를 각각 입력해 결과를 확인한다.
3. "시도명" 같은 명목형 열에는 평균 대신 `=COUNTA(범위)`로 개수만 세고, 평균을 내면 왜 무의미한지 한 줄로 적는다.
4. [삽입] > [피벗 테이블]로 명목형(시도)별 인구 합계/평균을 만들어, 범주형 변수가 어떻게 "그룹 요약"에 쓰이는지 본다.
5. [삽입] > 막대 그래프로 시도별 인구를 시각화한다. (범주형 = 막대, 연속형 분포 = 히스토그램이라는 짝을 기억한다.)

## 2. 데이터의 입력과 점검

### 개요: 왜 배우나

데이터 분석은 "쓰레기를 넣으면 쓰레기가 나온다(Garbage In, Garbage Out)"는 원칙이 지배한다. 아무리 좋은 분석 기법이나 멋진 그래프를 써도 입력된 원자료가 더럽다면 결론은 틀린다. 흔히 인용되는 추정치로, 분석가가 쓰는 시간의 60~80%는 모델링이 아니라 데이터를 읽어 들이고, 결측치와 이상치를 점검하고, 분석 가능한 형태로 정제(cleaning)하는 데 들어간다고 한다(정확히 측정된 수치라기보다 정제 작업의 비중이 그만큼 크다는 통념이다).

이 장에서 익히는 것은 "분석 전 단계"의 기초 체력이다. 즉, (1) 흩어진 원자료를 표(엑셀/CSV)로 불러오고, (2) 빠진 값과 튀는 값을 찾아내 처리하며, (3) 행=관측치 / 열=변수라는 정형 데이터의 약속을 지켜 분석 도구가 곧바로 먹을 수 있는 깔끔한 데이터(Tidy Data)로 만드는 일이다. 이 과정을 손에 익혀야 KOSIS나 공공데이터포털에서 받은 실제 데이터를 가지고 다음 장들(요약통계, 시각화, 회귀)로 나아갈 수 있다.

### 핵심 개념·기능

- 정형 데이터의 구조: 행(row)=관측 단위 1건(예: 사람 1명, 하루), 열(column)=변수 1개(예: 키, 소득), 셀(cell)=하나의 값. 이 약속이 모든 분석 도구의 전제다.
- 결측치(missing value): 값이 비어 있는 칸. 표기는 도구마다 다르다 - 엑셀 빈 칸, R의 `NA`, pandas의 `NaN`/`None`. 빈칸인지, 0인지, "모름"으로 입력된 문자열인지 반드시 구분해야 한다.
- 결측치 처리 전략: 행/열 삭제(`dropna`, `na.omit`), 대치(imputation) - 수치형은 평균·중앙값, 범주형은 최빈값으로 채우기(`fillna`), 모델 예측으로 채우기.
- 이상치(outlier): 분포에서 비정상적으로 멀리 떨어진 값. 입력 오류일 수도, 진짜 극단값일 수도 있다. 탐지법: 상자그림(boxplot), IQR 규칙(Q1-1.5×IQR 미만 또는 Q3+1.5×IQR 초과), Z-점수.
- 데이터 정제(cleaning): 중복 행 제거, 공백·오타 정리, 자료형(숫자/날짜/문자) 통일, 단위·표기 일관화, 인코딩(UTF-8) 깨짐 처리.
- Tidy Data 3원칙(Hadley Wickham): (1) 각 변수는 하나의 열, (2) 각 관측치는 하나의 행, (3) 각 값은 하나의 셀. "넓은 형식(wide)"을 "긴 형식(long)"으로 바꾸는 것이 핵심 변환이다.
- 데이터 첫 점검 루틴: 행·열 개수 확인 -> 앞/뒤 몇 줄 미리보기(`head`/`tail`) -> 자료형과 결측 현황 요약(`info`/`describe`, `str`/`summary`).

아래는 KOSIS 또는 공공데이터포털에서 실제 데이터를 받아 엑셀과 pandas/R로 점검하는 과정이다. 순서대로 직접 따라 한다.

실습 A - 실제 데이터 내려받기 (KOSIS / 공공데이터포털)

1. 브라우저에서 KOSIS(kosis.kr)에 접속한다. 상단 "국내통계 -> 주제별 통계"에서 관심 주제(예: 인구, 물가)를 고른다.
2. 통계표를 연 뒤 "항목/분류/시점"을 선택하고 조회한다. 화면 우측의 "다운로드" 버튼을 눌러 파일 형태를 EXCEL 또는 CSV로 골라 저장한다. (셀이 2만 개 이하면 조회 화면에서 바로 받을 수 있다.)
3. 비교용으로 공공데이터포털(data.go.kr)에서도 받아 본다. "파일데이터"는 로그인 없이 CSV 다운로드가 가능하다(OpenAPI만 회원가입·활용신청 필요). 검색 -> 원하는 데이터셋 -> "다운로드" 클릭.
4. 받은 CSV를 메모장으로 한 번 열어 한글이 깨지지 않는지(인코딩) 확인한다. 깨지면 엑셀에서 "데이터 -> 텍스트/CSV 가져오기"로 UTF-8 인코딩을 지정해 다시 연다.

실습 B - 엑셀로 점검·정제하기

1. 받은 파일을 엑셀로 연다. 1행이 변수명(헤더)인지, 위쪽에 제목·단위 같은 군더더기 행이 섞여 있는지 본다. 군더더기 행은 삭제해 "1행=헤더, 2행부터 데이터"로 만든다.
2. 결측치 찾기: 빈 칸을 한눈에 보려면 `Ctrl+A`로 전체 선택 -> "홈 -> 찾기 및 선택 -> 이동 옵션 -> 빈 셀"로 빈 셀을 강조한다. 또는 `=COUNTBLANK(범위)`로 빈 칸 개수를 센다.
3. 중복 제거: 데이터 영역을 선택하고 "데이터 -> 중복된 항목 제거"를 실행해 같은 행이 여러 번 들어간 것을 정리한다.
4. 이상치 눈으로 잡기: 한 수치 열을 선택해 "삽입 -> 통계 차트 -> 상자 수염(Box and Whisker)"을 그린다. 점으로 튀어나온 값이 이상치 후보다. 정렬(`Ctrl+Shift+L` 필터 -> 오름/내림차순)로 최솟값·최댓값을 직접 확인한다.
5. 결측치 채우기 연습: 한 수치 열의 평균을 `=AVERAGE(범위)`로 구해, 빈 칸에 그 값을 채워 본다(평균 대치). 원본은 다른 시트에 백업해 둔다.

실습 C - pandas(Python)로 점검·정제하기

1. 같은 CSV를 불러온다.

   ```python
   df.shape        # (행 수, 열 수)
   df.head()       # 앞 5줄
   df.info()       # 자료형 + 결측(non-null) 현황
   df.describe()   # 수치형 요약통계(min/max로 이상치 감지)
   ```
3. 결측치 현황을 센다.
   ```python
   df.isnull().sum()       # 열별 결측 개수
   ```
4. 결측치를 처리한다(두 가지를 다 해 보고 결과를 비교).
   ```python
   df2 = df.dropna()                          # 결측 있는 행 삭제
   df["소득"] = df["소득"].fillna(df["소득"].median())  # 중앙값으로 대치
   ```
5. 중복을 제거하고 이상치를 IQR 규칙으로 걸러 본다.
   ```python
   df = df.drop_duplicates()
   q1, q3 = df["소득"].quantile([0.25, 0.75])
   iqr = q3 - q1
   mask = (df["소득"] >= q1 - 1.5*iqr) & (df["소득"] <= q3 + 1.5*iqr)
   df_clean = df[mask]
   ```

## 3. 통계정보 검색 및 활용

> 교재 대응: 교재 3장 통계정보 검색 및 활용 KOSIS 통계표 검색-다운로드-기술통계·시각화 실습.

### 개요: 왜 배우나

데이터 분석은 "데이터를 어디서 구하느냐"에서 시작한다. 이 장은 신뢰할 수 있는 공공 데이터의 1차 공급처인 KOSIS(국가통계포털)에서 실제 통계를 직접 검색·다운로드하고, 그 데이터를 기술통계(descriptive statistics)로 요약·시각화하는 전 과정을 다룬다.

- 기술통계는 수백 수천 개의 숫자를 평균·중앙값·표준편차 같은 몇 개의 대표값으로 압축해, 데이터의 중심과 퍼짐, 모양을 한눈에 파악하게 해 준다.
- 실제 데이터(인구, 물가, 고용 등)를 다루면 교과서 예제와 달리 결측치·단위·정렬 문제를 만나게 되며, 이 경험이 이후 모든 데이터 과목의 토대가 된다.
- 이 장은 중간과제에 직결된다. 과제는 보통 "KOSIS에서 특정 통계표를 받아 기술통계 값을 구하고 그래프로 해석하라" 형태이므로, 검색-다운로드-요약-시각화 흐름을 손에 익혀야 한다.

### 핵심 개념·기능

- 중심 경향(대표값): 평균(mean), 중앙값(median), 최빈값(mode). 평균은 이상치에 민감하고 중앙값은 강건하다.
- 산포(퍼짐): 범위(range), 분산(variance), 표준편차(standard deviation), 사분위수(Q1/Q2/Q3)와 사분위범위(IQR = Q3 - Q1).
- 모집단 vs 표본 분산: 나누는 값이 n인지 n-1인지에 따라 다르다. 엑셀 `VAR.P`/`STDEV.P`(모집단) vs `VAR.S`/`STDEV.S`(표본), R `var()`/`sd()`(표본 기준), pandas `.var()`/`.std()`(기본 ddof=1, 표본 기준)를 구분해야 한다.
- 분포 시각화: 히스토그램(분포 모양·치우침), 상자그림(box plot, 사분위·이상치), 시계열 선그래프(시간에 따른 추세).
- KOSIS 핵심 기능: 통계 검색, 조회조건 설정(항목/분류/시점), 행렬 전환(시점을 행·열로), 파일 다운로드(EXCEL/CSV), 대용량(20,000셀 초과 시) 다운로드.

### 단계별 실습

#### 실습 A: KOSIS에서 실데이터 검색·다운로드

1. KOSIS(kosis.kr)에 접속해 상단 검색창에서 "통계표"를 선택하고 관심 주제(예: "시도별 인구", "소비자물가지수")를 입력한 뒤 검색한다.
2. 검색 결과 목록에서 원하는 통계표 제목을 클릭해 통계표 조회 화면으로 들어간다.
3. 화면 상단의 조회설정(항목 / 분류(시도 등) / 시점)을 연다. 시점 탭에서 분석할 기간(예: 최근 10년)을 체크하고, 항목·분류에서 필요한 값만 선택한 뒤 "조회"를 누른다.
4. 표가 나오면 행렬전환 기능으로 시점을 열에 두어 "지역 x 연도" 형태로 정돈한다(엑셀에서 다루기 쉬운 형태).
5. "다운로드" 버튼을 눌러 파일 형태를 EXCEL(.xlsx) 또는 CSV로 선택해 저장한다. (셀 20,000개를 초과하면 조회 버튼이 다운로드 버튼으로 바뀌고, 200,000개 초과 시에는 통계목록의 "파일다운로드"를 쓴다.)
6. 받은 파일을 열어 머리글 행, 단위(천 명 등), 빈 셀(결측치) 위치를 먼저 확인한다.

#### 실습 B: 엑셀로 기술통계 + 그래프

1. 다운로드한 파일을 엑셀에서 열고, 분석할 숫자 열 하나(예: 연도별 인구)를 정한다.
2. 빈 셀에 함수를 입력해 대표값을 구한다.
   - 평균: `=AVERAGE(B2:B11)`
   - 중앙값: `=MEDIAN(B2:B11)`
   - 표본 표준편차: `=STDEV.S(B2:B11)` / 표본 분산: `=VAR.S(B2:B11)`
   - 사분위: `=QUARTILE.INC(B2:B11,1)`(Q1), `=QUARTILE.INC(B2:B11,3)`(Q3)
3. 한 번에 보려면 [데이터] 탭 > [데이터 분석] > "기술 통계법"을 실행한다. (안 보이면 [파일 > 옵션 > 추가 기능 > Excel 추가 기능 > 분석 도구]를 체크해 활성화한다.)
4. 데이터 범위를 선택하고 [삽입] > 차트에서 히스토그램(분포 확인)과 꺾은선형(시계열 추세)을 각각 그린다. 상자그림은 [삽입 > 통계 차트 > 상자 수염]으로 만든다.
5. 그래프 제목·축 레이블·단위를 달고, "평균보다 중앙값이 낮다 = 오른쪽으로 꼬리가 긴 분포" 식으로 한 줄 해석을 적는다.

#### 실습 C: R 또는 Python으로 같은 데이터 요약(확장)

1. R: `data <- read.csv("kosis.csv")` 로 불러온 뒤 `summary(data$인구)`, `mean()`, `median()`, `sd()`, `quantile()`로 값을 구하고 `hist()`, `boxplot()`, `plot(type="l")`로 시각화한다.
2. Python(pandas): `import pandas as pd; df = pd.read_csv("kosis.csv")` 후 `df["인구"].describe()` 한 줄로 count·mean·std·min·사분위·max를 한꺼번에 본다. 시각화는 `df["인구"].plot(kind="hist")`, `df.boxplot()`, `df["인구"].plot()`.
3. 엑셀 결과와 R/Python 결과의 표준편차가 미세하게 다르면, 표본(n-1) vs 모집단(n) 차이인지 확인한다.

## 4. 문서작성

> 교재 대응: 교재 4장 문서작성/한글 2018 수식편집기·연습문제. 교재 화면·메뉴는 한글 2018 기준이다(최신 버전에서는 메뉴 위치·명칭이 조금 다를 수 있으니 같은 기능을 찾아 대응한다).

### 개요: 왜 배우나

데이터 분석은 "분석으로 끝나지 않고, 남이 읽고 판단할 수 있는 문서로 전달될 때 비로소 가치가 생긴다." 아무리 좋은 분석 결과라도 표가 어그러지거나, 수식이 그림처럼 깨져 보이거나, 글의 흐름이 뒤죽박죽이면 읽는 사람은 결론을 신뢰하지 않는다. 이 장은 분석 결과를 한글(HWP)/워드 같은 워드프로세서에서 **정확한 표**, **올바른 수식**, **논리적인 보고서 구조(서론-분석-결론)**로 정리하는 능력을 기르는 것이 목표다.

1학년 1학기 입문 과목에서 이 장을 배우는 이유는, 앞으로 모든 과목의 과제·TMA·실습 보고서가 결국 "문서"로 제출되기 때문이다. 표/수식/구조를 지금 손에 익혀두면 4년 내내 쓴다.

### 핵심 개념·기능

- **표(Table) 편집**
  - 셀 병합/분할, 행·열 추가·삭제, 셀 크기 자동 맞춤
  - 셀 테두리/배경, 제목 행 음영, 셀 정렬(숫자는 오른쪽 정렬·소수점 자리 통일)
  - 표 안 계산식: 한글에서는 셀 이름을 소괄호로 묶고 셀 사이를 쉼표(,) 또는 콜론(:)으로 구분 (예: `=SUM(A1:A5)`)
  - 표 제목은 표 **위**, 그림 제목은 그림 **아래**에 다는 관행
- **시계열 차트(분산형 X,Y)**: 연도·시점이 가로축인 시계열을 보고서에 넣을 때는 교재 예제처럼 분산형(X,Y) 차트로 그린다. 엑셀에서 [삽입] > [분산형] > "직선 및 표식이 있는 분산형"을 고르면 X축에 연도, Y축에 값이 들어간 시계열 그래프가 된다(꺾은선과 달리 X축 간격이 실제 수치 간격으로 반영된다).
- **수식 편집기(Equation Editor)**
  - 한글: 입력 메뉴 > 수식, 단축키 `Ctrl + N, M`
  - 분수, 첨자(위/아래 첨자), 루트, 시그마(∑), 적분, 행렬 등 구조 입력
  - 수식 글자 크기는 일반 본문처럼 블록 지정으로 바뀌지 않고 수식 편집기 안에서 따로 조정해야 함
- **분석 보고서 구조 (서론-분석-결론)**
  - 서론: 주제·배경·목적, 사용한 데이터 출처(예: KOSIS), "무엇을 왜 분석하는가"
  - 분석(본론): 데이터 설명 -> 분석 방법 -> 표·그래프 -> 결과 해석
  - 결론: 핵심 발견 요약 + 시사점/제안 + 한계·향후 과제
  - 표·그림에는 일련번호(<표 1>, [그림 1])와 캡션, 출처를 반드시 표기

### 단계별 실습

#### 실습 A. KOSIS에서 데이터 받아 표로 정리하기

1. KOSIS 국가통계포털(kosis.kr)에 접속해 관심 주제(예: "시도별 인구")를 검색한다.
2. 통계표 조회 화면에서 항목/분류/시점을 고른 뒤, 다운로드 버튼을 눌러 **EXCEL 또는 CSV**로 내려받는다.
3. 엑셀에서 불필요한 행/열을 지우고, 분석에 쓸 5~10개 행만 남긴다.
4. 정리한 표 범위를 복사한다.
5. 한글/워드를 열고 표를 붙여넣되, "원본 서식 유지"가 아니라 "표로 붙여넣기"를 선택해 워드프로세서 표로 변환한다.
6. 제목 행에 음영을 주고, 숫자 열은 모두 오른쪽 정렬 + 소수점 자리수를 통일한다.
7. 표 위에 `<표 1> 시도별 인구 현황(2024)` 형식의 제목을, 표 아래에 `출처: KOSIS` 를 단다.

#### 실습 B. 표 안에서 합계 자동 계산하기 (한글 기준)

1. 한글에서 3행 x 4열 표를 만든다(입력 > 표 > 표 만들기, `Ctrl + N, T`).
2. 맨 위 행에 항목명, 그 아래 행들에 숫자를 입력한다.
3. 합계를 넣을 빈 셀에 커서를 두고 표 메뉴의 계산식(또는 셀 블록 후 자동 합계)을 연다.
4. `=SUM(B2:B4)` 처럼 셀 범위를 지정해 합계를 넣는다.
5. 원본 숫자를 바꾼 뒤 계산식을 다시 계산해, 값이 갱신되는지 확인한다.

#### 실습 C. 수식 편집기로 평균·표준편차 식 쓰기

1. 한글에서 `Ctrl + N, M`으로 수식 편집기를 연다(워드는 삽입 > 수식).
2. 분수 틀을 골라 평균 식을 만든다: 분자에 `x_1 + x_2 + ... + x_n`, 분모에 `n`.
3. 시그마 기호를 불러와 합 형태로 바꿔본다: `bar x = {1 over n} sum_{i=1}^{n} x_i`.
4. 루트와 분수를 조합해 표준편차 식을 만든다: `s = sqrt{ {1 over {n-1}} sum (x_i - bar x)^2 }`.
5. 수식이 너무 작으면 수식 편집기 안에서 글자 크기를 키운 뒤 본문으로 빠져나온다.

#### 실습 D. 한 장짜리 분석 보고서(서론-분석-결론) 조립하기

1. 새 문서를 열고 제목과 작성자/날짜를 맨 위에 쓴다.
2. **서론**(3~5줄): 분석 주제, 왜 궁금했는지, 어떤 데이터(KOSIS 등)를 썼는지 적는다.
3. **분석**: 실습 A에서 만든 표(<표 1>)와 실습 C의 수식을 본문에 배치하고, 표 바로 아래에 "표를 보면 ~한 경향이 나타난다"처럼 **해석 문장**을 붙인다.
4. 가능하면 표 데이터를 차트로 만들어 [그림 1]로 넣고, 그림 아래에 캡션을 단다.
5. **결론**(3~5줄): 핵심 발견 1~2개를 요약하고, 시사점 또는 다음에 더 분석할 것(한계·향후 과제)을 쓴다.
6. 표 정렬·캡션·맞춤법을 최종 점검하고 PDF로 내보내 레이아웃이 깨지지 않는지 확인한다.

## 5. 엑셀 데이터 처리

### 개요 (왜 배우나)

데이터 분석의 출발점은 거창한 프로그래밍 언어가 아니라 엑셀이다. KOSIS나 공공데이터포털에서 받은 통계는 대부분 엑셀(.xlsx)이나 CSV로 내려받게 되고, R이나 Python으로 넘기기 전에 "데이터를 눈으로 보고 정리하는" 일이 반드시 선행된다. 엑셀은 코드 없이도 셀 참조, 함수, 정렬/필터/피벗만으로 집계와 요약 통계를 즉시 뽑아낼 수 있어, 데이터 분석의 사고방식(원본 데이터 -> 가공 -> 요약 -> 해석)을 가장 빠르게 익히는 도구다. 이 장에서 배우는 절대/상대 참조, 조건부 집계 함수, 피벗테이블은 이후 R의 `dplyr`이나 Python `pandas`의 `groupby`로 그대로 이어지는 개념이라, 잘 다져두면 뒤 학기 프로그래밍 과목이 훨씬 수월해진다.

### 핵심 개념·기능

- **셀 참조**
    - 상대 참조 (`A1`): 수식을 복사하면 참조가 함께 이동한다.
    - 절대 참조 (`$A$1`): 행·열을 모두 고정. 복사해도 위치가 안 변한다.
    - 혼합 참조 (`$A1`, `A$1`): 행 또는 열만 고정. `F4` 키를 반복해서 누르면 `A1` -> `$A$1` -> `A$1` -> `$A1` 순으로 전환된다.
- **조회 함수**
    - `VLOOKUP(찾을값, 범위, 열번호, 일치옵션)`: 표의 첫 열에서 값을 찾아 같은 행의 다른 열 값을 가져온다. 일치옵션은 실무에서 거의 항상 `FALSE`(0, 정확히 일치)를 쓴다. 범위는 `F4`로 절대 참조 고정이 필수.
- **논리·조건 집계 함수**
    - `IF(조건, 참일때, 거짓일때)`: 조건 분기.
    - `COUNTIF(범위, 조건)`: 조건을 만족하는 셀 개수.
    - `SUMIF(조건범위, 조건, 합계범위)`: 조건을 만족하는 행의 값 합계.
    - 확장판으로 다중 조건을 받는 `COUNTIFS`, `SUMIFS`, `AVERAGEIF(S)`가 있다.
- **데이터 정리 기능**
    - 정렬: 한 개 또는 여러 기준(예: 지역 오름차순 -> 매출 내림차순)으로 행 재배열.
    - 필터: 조건에 맞는 행만 표시. 자동 필터, 슬라이서.
    - 피벗테이블: 행/열/값 영역에 필드를 끌어 넣어 즉석 교차 집계. 데이터 분석의 핵심 도구.
- **통계 함수**
    - `AVERAGE(범위)`: 산술 평균.
    - `STDEV.S(범위)` / `STDEV.P(범위)`: 표본 표준편차 / 모집단 표준편차. 구버전 함수와 한 번 더 못박아 두면, **`STDEV` = 표본(n-1로 나눔) = `STDEV.S`**, **`STDEVP` = 모집단(n으로 나눔) = `STDEV.P`**다. 이름 끝의 `P`(Population)가 모집단이라는 신호이며, `P`가 없는 쪽이 표본임을 기억한다(분산도 같은 규칙: `VAR`/`VAR.S`=표본, `VARP`/`VAR.P`=모집단).
    - `CORREL(범위1, 범위2)`: 두 변수의 상관계수(-1 ~ +1).
    - 함께 알아두면 좋은 것: `MEDIAN`(중앙값), `MAX`/`MIN`, `COUNT`.

### 단계별 실습

미리 KOSIS 또는 공공데이터포털에서 엑셀/CSV 파일 하나를 내려받아 둔다(아래 미션 참고). 없으면 직접 표를 만들어 따라 해도 된다.

**실습 A - 셀 참조와 조건 함수 (성적/매출 표 만들기)**

1. 새 시트에 `이름`, `점수` 두 열로 10행 정도의 표를 입력한다(점수는 임의 숫자).
2. `C2` 셀에 `=IF(B2>=60, "합격", "불합격")`을 입력하고 채우기 핸들로 아래로 끌어 복사한다.
3. 빈 셀에 `=COUNTIF(B2:B11, ">=60")`을 입력해 합격자 수를 센다.
4. 같은 식으로 `=SUMIF(B2:B11, ">=60")`을 입력해 합격자 점수 합계를 구한다.
5. `=AVERAGE(B2:B11)`, `=MEDIAN(B2:B11)`, `=STDEV.S(B2:B11)`을 각각 입력해 평균·중앙값·표준편차를 비교한다.

**실습 B - 절대 참조와 VLOOKUP (코드표 조회)**

1. 두 번째 표로 `등급`, `장학금` 같은 코드표를 만든다(예: A행=A등급/30만, B행=B등급/20만).
2. 첫 표 옆에 `등급` 열을 만들고, 각 점수에 맞는 등급을 입력한다.
3. `장학금` 열의 첫 셀에 `=VLOOKUP(등급셀, 코드표범위, 2, FALSE)`을 입력한다.
4. 수식 입력 중 코드표 범위를 드래그한 직후 `F4`를 눌러 `$A$1:$B$3`처럼 절대 참조로 고정한다.
5. 채우기 핸들로 아래까지 복사하고, 모든 행이 올바른 장학금을 가져오는지 확인한다(고정 안 하면 범위가 밀려 `#N/A`가 난다 - 일부러 한 번 깨뜨려 보고 원리를 체감한다).

**실습 C - 정렬·필터**

1. 표 안 아무 셀이나 클릭한 뒤 [데이터] 탭 -> [필터]를 켠다.
2. `점수` 열 필터 화살표에서 "숫자 필터 -> 보다 큼" 으로 60 초과만 표시해 본다.
3. [데이터] -> [정렬]에서 1순위 기준을 `등급`(오름차순), 2순위를 `점수`(내림차순)로 지정해 다중 정렬을 적용한다.
4. 필터를 해제해 원본 전체가 복구되는지 확인한다.

**실습 D - 피벗테이블**

1. 데이터 표 안을 클릭 -> [삽입] 탭 -> [피벗 테이블] -> "새 워크시트" 로 생성한다.
2. 오른쪽 필드 목록에서 `등급`을 **행** 영역으로, `점수`를 **값** 영역으로 끌어 넣는다.
3. 값 영역의 `합계: 점수`를 클릭 -> "값 필드 설정" -> 계산을 **평균**으로 바꾼다(개수, 합계로도 바꿔 보며 차이 관찰).
4. `이름` 등 다른 필드를 **필터** 영역에 넣어 특정 조건만 보는 슬라이서 효과를 만들어 본다.
5. 같은 결과를 함수(`AVERAGEIF`)로도 계산해 보고 피벗 결과와 일치하는지 대조한다.

**실습 E - 상관관계 보기**

1. 두 개의 수치 열(예: `공부시간`, `점수`)을 가진 표를 만든다.
2. 빈 셀에 `=CORREL(공부시간범위, 점수범위)`를 입력해 상관계수를 구한다.
3. 두 열을 선택 -> [삽입] -> 분산형(산점도) 차트를 그려, 상관계수 부호·크기와 점들의 흩어진 모양을 눈으로 연결해 본다.

## 6. 엑셀 그래프 그리기

### 개요 - 왜 배우나

표(table)에 숫자가 가득 차 있어도 사람의 눈은 숫자 덩어리에서 패턴을 바로 읽어내지 못한다. 그래프는 데이터를 시각 신호(길이, 위치, 기울기, 점의 흩어짐)로 바꿔서 "무엇이 크고, 어떻게 변하고, 둘 사이에 관계가 있는가"를 한눈에 보이게 한다. 데이터 분석 입문에서 그래프 그리기를 배우는 이유는 세 가지다.

- 분석 결과를 남(교수, 동료, 보고서 독자)에게 설득력 있게 전달하기 위해서.
- 분석을 시작하기 전에 데이터의 분포와 이상치(outlier)를 눈으로 먼저 탐색(EDA, 탐색적 데이터 분석)하기 위해서.
- 데이터의 성격(범주형 vs 수치형, 시간 흐름 vs 두 변수 관계)에 맞는 차트를 고르는 판단력을 기르기 위해서. 잘못된 차트는 데이터를 왜곡한다.

이 과목은 실습 과목이므로, 개념을 읽는 데서 그치지 말고 엑셀을 직접 열어 KOSIS에서 받은 실제 통계로 차트를 만들어 보는 것이 핵심이다.

### 핵심 개념과 기능

- **차트 종류 선택 기준 (데이터 성격 = 차트 종류)**
    - **원형(파이) 차트**: 전체에 대한 각 항목의 구성비(점유율)를 보여줄 때. 항목이 적고(5개 이내 권장) 한 항목이 두드러질 때 효과적. 합이 100%인 데이터에만 쓴다.
    - **막대 차트(세로/가로)**: 여러 항목 간 값의 크기를 비교할 때. 세로막대는 항목 10개 이내, 항목 이름이 길거나 개수가 많으면 가로막대가 읽기 좋다.
    - **꺾은선 차트**: 시간 흐름(월, 분기, 연도)에 따른 추세(증가/감소)를 볼 때. 시계열 데이터의 기본.
    - **히스토그램**: 하나의 수치형 변수가 어떤 구간(계급)에 얼마나 몰려 있는지 분포를 볼 때. 막대 차트와 달리 가로축이 "구간"이고 막대 사이가 붙어 있다.
    - **산점도(분산형)**: 두 개의 수치형 변수 사이의 관계를 점으로 찍어 볼 때.
- **산점도와 상관관계**
    - 점들이 우상향으로 모이면 양의 상관, 우하향이면 음의 상관, 흩어져 있으면 무상관.
    - **추세선(trendline)**: 점들에 가장 잘 맞는 직선/곡선. 관계의 방향과 세기를 요약한다.
    - **상관계수와 R제곱(결정계수)**: R제곱이 1에 가까울수록 추세선이 데이터를 잘 설명한다. 단, 상관관계는 인과관계가 아니다(상관이 있어도 원인-결과로 단정 금지).
- **차트 구성 요소(읽기 쉬운 그래프의 조건)**
    - **축(axis)**: 가로축(범주/X)과 세로축(값/Y). 축의 시작값을 0이 아닌 곳에서 끊으면 차이가 과장되어 보이므로 주의.
    - **축 제목·눈금 단위**: 무엇을, 어떤 단위(명, 원, %)로 재는지 명시.
    - **범례(legend)**: 데이터 계열이 둘 이상일 때 색/모양이 무엇을 뜻하는지 표시.
    - **차트 제목**: 그래프가 말하려는 핵심 메시지를 한 줄로.

### 단계별 실습

준비물: 엑셀(또는 무료 대안인 LibreOffice Calc / Google Sheets), 인터넷.

**실습 A - KOSIS 데이터로 막대·꺾은선 차트 만들기**

1. KOSIS(kosis.kr)에 접속해 관심 주제(예: "시도별 인구" 또는 "연도별 1인당 국민총소득")를 검색한다.
2. 통계표를 연 뒤 [다운로드]에서 형식을 Excel(.xlsx)로 받아 저장한다.
3. 엑셀에서 파일을 열고, 차트로 만들 범위(항목 이름 열 + 값 열)를 드래그해 선택한다. 머리글(제목 행)까지 포함해 선택하면 축 이름이 자동으로 들어간다.
4. 시도별 비교라면 [삽입] 탭 → [세로 또는 가로 막대형 차트], 연도별 변화라면 [삽입] → [꺾은선형 차트]를 누른다. 어떤 차트가 좋을지 모르면 [삽입] → [추천 차트]를 눌러 엑셀이 제안하는 후보를 본다.
5. 생성된 차트를 클릭하면 우측 상단에 나타나는 [+] (차트 요소) 버튼으로 **축 제목**, **차트 제목**, **범례**를 켜고 끈다.
6. 차트 제목을 더블클릭해 "메시지가 담긴 제목"으로 바꾼다. 예: "그래프1"이 아니라 "2015-2024년 1인당 국민총소득 증가 추이".

**실습 B - 산점도와 추세선으로 상관관계 보기**

1. 두 수치형 변수를 두 열에 나란히 입력한다(예: A열 = 광고비, B열 = 매출). KOSIS의 두 통계표를 연도 기준으로 합쳐도 좋다.
2. 두 열을 모두 선택한 뒤 [삽입] → [분산형(산점도)] 차트를 고른다.
3. 점들이 우상향/우하향/무작위 중 어떤 모양인지 눈으로 먼저 판단한다.
4. 점 하나에 마우스 오른쪽 클릭 → [추세선 추가]를 선택한다.
5. 추세선 옵션에서 **선형**을 고르고, 아래쪽의 **수식을 차트에 표시**와 **R-제곱 값을 차트에 표시**를 체크한다.
6. 표시된 R제곱 값을 확인한다. 1에 가까우면 두 변수의 직선 관계가 강하다는 뜻. 이 결과를 한 문장으로 해석해 메모한다(예: "광고비가 늘수록 매출이 늘어나는 양의 상관이 보이며 R제곱은 0.8로 설명력이 높다").

**실습 C - 히스토그램으로 분포 보기**

1. 한 변수의 값을 한 열에 길게 입력하거나 붙여넣는다(예: 학생 50명의 시험 점수).
2. 그 열을 선택하고 [삽입] → [통계 차트] → [히스토그램]을 누른다.
3. 가로축(구간)을 더블클릭 → [축 서식]에서 **계급 구간(bin) 너비** 또는 개수를 바꿔 가며 분포 모양이 어떻게 달라지는지 관찰한다.
4. 어떤 구간에 값이 가장 몰려 있는지(최빈 구간), 한쪽으로 치우쳤는지(왜도)를 메모한다.

**실습 D - 잘못된 차트 바로잡기(왜곡 방지)**

### 단순선형회귀 (산점도 추세선의 정체)

실습 B에서 산점도에 그린 "선형 추세선"이 바로 **단순선형회귀(simple linear regression)**의 결과다. 추세선이라는 시각적 도구 뒤에 어떤 수식이 있는지 한 단계만 더 들여다보면, 두 변수의 관계를 숫자로 요약할 수 있다.

- **회귀식**: `y = a + bx`
    - `x`(독립변수, 원인 후보) 하나로 `y`(종속변수, 결과)를 직선으로 예측하는 식이다.
    - **기울기 `b`**: `x`가 1 늘 때 `y`가 평균적으로 얼마나 변하는지를 나타낸다(관계의 방향과 세기). `b > 0`이면 우상향(양의 관계), `b < 0`이면 우하향(음의 관계).
    - **절편 `a`**: `x = 0`일 때의 `y` 예측값. 데이터 범위에 `x = 0`이 없으면 해석상 의미가 약할 수 있으니 "직선이 세로축과 만나는 위치"로만 받아들인다.
- **R제곱(R², 결정계수)**: 회귀직선이 `y`의 흩어짐(분산) 중 몇 %를 설명하는지를 0~1로 나타낸다. 1에 가까울수록 점들이 직선에 잘 붙어 있어 예측력이 높다. 단순선형회귀에서는 R제곱이 상관계수 `r`의 제곱과 같다. (R제곱이 높아도 인과관계를 뜻하지는 않는다.)

**엑셀로 회귀식·R제곱 확인하기**: 실습 B의 산점도에서 점 위 마우스 오른쪽 클릭 → [추세선 추가] → **선형** 선택 후, 추세선 옵션 아래쪽의 **수식을 차트에 표시**와 **R-제곱 값을 차트에 표시**를 체크한다. 그러면 차트에 `y = bx + a` 형태의 회귀식과 `R² = ...` 값이 함께 표시된다. 함수로 직접 구하려면 `=SLOPE(y범위, x범위)`(기울기 b), `=INTERCEPT(y범위, x범위)`(절편 a), `=RSQ(y범위, x범위)`(R제곱)를 쓴다.

**R로 한 줄 회귀**: R에서는 `lm()`(linear model)으로 회귀모형을 적합하고 `summary()`로 계수·R제곱을 확인한다.

```r
fit <- lm(y ~ x, data = df)   # y를 x로 설명하는 단순선형회귀 적합
summary(fit)                  # 절편(Intercept)=a, x의 기울기=b, Multiple R-squared=R제곱
```

`summary(fit)` 출력에서 `Coefficients`의 `(Intercept)` 행이 절편 `a`, `x` 행이 기울기 `b`이고, 아래쪽 `Multiple R-squared` 값이 R제곱이다. 엑셀 추세선이 보여 준 수식·R제곱과 같은 값이 나오는지 대조해 보면 "도구가 바뀌어도 결과는 같다"는 감각을 굳힐 수 있다.

## 7. 엑셀 활용

### 개요: 왜 배우나

데이터 분석은 "깨끗하게 정리된 표"에서 출발하지만, 현실 데이터는 그렇지 않다. 날짜가 `20240315` 같은 숫자나 `2024-03-15` 텍스트로 들어오고, 이름·주민번호·주소가 한 칸에 뒤섞여 있으며, "월 적립 30만원을 3년 부으면 만기에 얼마인가" 같은 의사결정 질문이 끊임없이 따라온다. 7장은 엑셀을 단순 표 계산기가 아니라 **전처리 + 의사결정 + 시각화 도구**로 쓰는 법을 다룬다.

- 날짜/시간 함수: 통계 데이터의 시계열 정리(연·월·분기 추출, 기간 계산)의 기본기
- 문자열 함수: KOSIS·공공데이터에서 받은 지저분한 텍스트 셀을 분석 가능한 열로 쪼개는 도구
- 원리합계(재무함수): 이자·적금·대출처럼 "시간에 따라 불어나는 값"을 수식 하나로 계산
- 해 찾기(Solver): "결과를 이 값으로 만들려면 입력을 얼마로?"라는 역방향·최적화 문제 해결
- 수식 함수 그래프: 수식과 데이터를 분산형/꺾은선 차트로 시각화해 추세를 눈으로 확인

이 네 가지는 뒤에 배울 R/Python 분석의 사전 감각(전처리, 시뮬레이션, 시각화)을 손으로 미리 익히는 단계다.

### 핵심 개념·기능

**1) 날짜/시간 함수**
- `TODAY()`, `NOW()`: 오늘 날짜 / 현재 날짜+시각
- `YEAR(날짜)`, `MONTH(날짜)`, `DAY(날짜)`: 날짜에서 연·월·일 추출
- `DATE(연,월,일)`: 숫자 3개를 진짜 날짜값으로 조립
- `EDATE(시작일, 개월)`: N개월 뒤/전 날짜, `EOMONTH(시작일, 개월)`: 해당 월의 말일
- `DATEDIF(시작, 끝, "Y"/"M"/"D")`: 두 날짜 사이 햇수/개월수/일수 (근속·나이 계산)
- `WEEKDAY`, `NETWORKDAYS`: 요일, 휴일 제외 근무일수
- 핵심 원리: 엑셀에서 날짜는 1900-01-01을 1로 세는 **일련번호(숫자)**다. 그래서 `날짜 - 날짜 = 일수`가 그냥 뺄셈으로 된다.

**2) 문자열(텍스트) 함수**
- `LEFT(텍스트, n)`, `RIGHT(텍스트, n)`: 왼쪽/오른쪽에서 n글자
- `MID(텍스트, 시작위치, n)`: 중간 특정 위치부터 n글자
- `LEN(텍스트)`: 글자 수, `FIND/SEARCH(찾을값, 텍스트)`: 위치 번호(FIND는 대소문자 구분)
- `&` 또는 `CONCAT`: 문자열 연결, `TRIM`: 불필요한 공백 제거
- `TEXT(값, 서식)`: 숫자/날짜를 원하는 형식 문자열로
- 핵심 패턴: `LEFT/MID` + `FIND`를 조합하면 길이가 제각각인 셀에서도 "구분자 기준"으로 자를 수 있다 (예: 이메일에서 `@` 앞만 추출).

**3) 원리합계(재무함수)**
- 원금에 이자가 붙고, 그 이자에 다시 이자가 붙는 **복리** 계산
- `FV(이율, 기간, 정기납입액, [현재가치], [납입시점])`: 미래가치(적금 만기금액)
- `PV(이율, 기간, 정기납입액, [미래가치])`: 현재가치
- `PMT(이율, 기간, 현재가치, [미래가치])`: 매기 납입/상환액(대출 월상환금, 적금 월납입금)
- 부호 규칙: 내가 내는 돈은 음수(-), 받는 돈은 양수(+). 월 단위면 이율은 `연이율/12`, 기간은 `년수*12`.

**4) 해 찾기 / 목표값 찾기**
- 목표값 찾기(Goal Seek): 변수 **1개**를 바꿔 수식 결과를 특정 값으로 (예: 목표 만기금액을 맞추는 월 납입액)
- 해 찾기(Solver): 변수 **여러 개** + 제약조건 하에서 목표를 최대/최소/특정값으로 만드는 **최적화** 도구 (추가 기능이라 별도 로드 필요)
- 구성 요소: 목표 셀(Objective), 변수 셀(Changing cells), 제한 조건(Constraints)

**5) 수식 함수 그래프**
- x 값들을 열에 채우고 → y = f(x) 수식 열을 만들고 → 두 열을 선택해 **분산형(곡선 포함)** 또는 꺾은선 차트로 그림
- 분산형(Scatter)은 x·y 모두 수치일 때, 꺾은선(Line)은 x가 범주(월·연도 등)일 때 적합

**실습 D. 해 찾기/목표값 찾기로 역산**
1. 먼저 추가 기능 로드: `파일 > 옵션 > 추가 기능 > 관리: Excel 추가 기능 [이동] > "해 찾기 추가 기능" 체크 > 확인`. 그러면 `데이터` 탭 오른쪽에 [해 찾기]가 생긴다.
2. (간단판, 목표값 찾기) 실습 C의 적금 셀을 재사용한다. 만기 목표가 1,200만원이라고 하자. `데이터 > 가상 분석 > 목표값 찾기` 선택.
3. 수식 셀=`B1`, 찾는 값=`12000000`, 값을 바꿀 셀=`A3`(월납입) 지정 후 확인 → 목표 만기를 맞추는 월 납입액이 자동으로 계산된다.
4. (Solver판) 목표 셀=`B1`, 목표=지정값 `12000000`, 변수 셀=`A3`로 두고, 제한 조건에 `A3 <= 400000`(월 40만원 한도) 같은 조건을 추가한 뒤 [해 찾기] 실행 → 제약 안에서 해를 찾는다.
5. 결과를 "값 유지"로 두고, 제약을 바꿔가며 해가 어떻게 달라지는지 본다.

**실습 E. 수식 y = f(x) 그래프 그리기**
1. A1에 `x`, B1에 `y`. A2에 `-5` 입력, A3에 `=A2+0.5` 입력 후 A3을 아래로 드래그해 `5`까지 채운다(x를 -5~5, 0.5 간격).
2. B2에 `=A2^2` 입력(이차함수 y=x^2) 후 아래로 채운다. (다른 수식으로 `=SIN(A2)` 등도 시도 가능)
3. A1:B(끝) 범위를 선택 → `삽입 > 차트 > 분산형(Scatter) > 곡선이 있는 분산형` 선택.
4. 차트 제목·축 제목을 추가하고, x=0 부근에서 곡선의 모양(포물선)을 확인한다.
5. B열 수식을 `=A2^3`이나 `=EXP(A2)`로 바꿔 그래프가 실시간으로 변하는 것을 관찰한다.

**실습 F. (연계) KOSIS 실제 데이터로 종합 적용**
1. KOSIS(국가통계포털)에 접속 → 관심 주제(예: 인구, 물가) 통계표 조회.
2. 다운로드 아이콘 클릭 → 팝업에서 **EXCEL** 형태 선택 후 다운로드(무료, 회원가입 불필요).
3. 받은 파일에서 날짜/연도 열을 `YEAR`/`MONTH`로 정리하고, 텍스트가 섞인 열을 `LEFT/MID`로 정돈한다.
4. 정돈한 시계열을 꺾은선 차트로 그려 추세를 확인한다(실습 A·B·E의 종합).

## 8. R을 활용한 자료분석

### 개요: 왜 배우나

엑셀은 직관적이지만 데이터가 커지거나 같은 분석을 반복할 때 한계가 있다. R은 통계 분석과 데이터 시각화를 위해 만들어진 무료 오픈소스 언어로, 코드 한 줄로 수천 행을 처리하고 분석 과정을 기록해 그대로 재현할 수 있다. 이 장에서 R과 RStudio를 설치하고, 데이터를 담는 그릇인 벡터와 데이터프레임을 익히며, `mean()` 같은 기본 통계함수와 `plot()` 시각화까지 다룬다. 1학년 단계에서 R의 기본기를 잡아 두면 이후 '오픈소스기반데이터분석', '머신러닝', '딥러닝' 같은 상위 과목으로 자연스럽게 이어진다. 엑셀(7장)과 같은 데이터를 R로도 다뤄 보면서 "도구를 바꿔도 분석 사고는 같다"는 감각을 기르는 것이 핵심 목표다.

### 핵심 개념·기능

- **R과 RStudio의 관계**: R은 실제 계산을 수행하는 엔진(언어), RStudio는 코드 작성·실행·그래프 확인을 한 화면에서 돕는 통합개발환경(IDE)이다. 보통 R을 먼저 설치하고 RStudio를 설치한다.
- **RStudio 4분할 화면**: 소스 편집기(좌상), 콘솔(좌하), 환경/히스토리(우상), 파일/플롯/패키지/도움말(우하).
- **할당 연산자 `<-`**: `x <- 10` 처럼 값을 변수에 저장한다. `=`도 되지만 R에서는 `<-`가 관행이다.
- **벡터(vector)**: 같은 자료형 값을 1차원으로 묶은 가장 기본 구조. `c(1, 2, 3)` 으로 만든다. 숫자형·문자형·논리형이 있다.
- **데이터프레임(data.frame)**: 행과 열로 된 표 형태 구조로 엑셀 시트와 가장 닮았다. 열마다 자료형이 다를 수 있다.
- **기본 통계함수**: `mean()`(평균), `median()`(중앙값), `sd()`(표준편차), `min()`/`max()`, `sum()`, `summary()`(요약통계 한 번에), `table()`(빈도표).
- **시각화 함수**: `plot()`(산점도·기본 그래프), `hist()`(히스토그램), `boxplot()`(상자그림), `barplot()`(막대그래프).
- **데이터 불러오기**: `read.csv("파일.csv")` 로 CSV를 데이터프레임으로 읽어 들인다.
- **패키지(package)**: 기능 묶음. `install.packages("이름")` 으로 설치하고 `library(이름)` 으로 불러온다.
- **도움말**: `?mean` 또는 `help(mean)` 로 함수 사용법을 즉시 확인.

### 단계별 실습

#### 실습 A. 설치와 첫 코드 실행 (설치 부담 없이 시작하려면 Posit Cloud로 대체 가능)

1. 브라우저에서 CRAN(https://cran.r-project.org)에 접속해 본인 OS(Windows/Mac)에 맞는 R을 내려받아 설치한다.
2. 이어서 Posit 다운로드 페이지(https://posit.co/downloads)에서 무료 **RStudio Desktop**을 내려받아 설치한다.
3. (설치가 번거롭다면) https://posit.cloud 에 가입해 브라우저에서 바로 RStudio를 쓴다. 무료 플랜은 월 25시간까지 사용 가능하다.
4. RStudio를 열고 좌하단 **콘솔**에 `1 + 1` 을 입력 후 Enter. `[1] 2` 가 나오면 정상이다.
5. `print("Hello R")` 을 실행해 출력 동작을 확인한다.

#### 실습 B. 벡터로 기본 통계 내기

1. 콘솔에 다음을 입력해 점수 벡터를 만든다: `scores <- c(85, 90, 78, 92, 88)`
2. 우상단 **Environment** 창에 `scores`가 보이는지 확인한다(저장 성공).
3. 평균과 표준편차를 구한다: `mean(scores)`, `sd(scores)`
4. `summary(scores)` 로 최소·1사분위·중앙값·평균·3사분위·최대를 한 번에 본다.
5. `max(scores)`, `min(scores)`, `length(scores)`(원소 개수)도 차례로 실행해 결과를 노트에 적는다.

#### 실습 C. 데이터프레임 만들고 다루기

1. 학생 표를 만든다:
   `df <- data.frame(name = c("가","나","다"), score = c(85, 90, 78))`
2. `df` 를 실행해 표 형태로 출력되는지 확인한다.
3. `str(df)` 로 각 열의 자료형을, `head(df)` 로 앞부분을 본다.
4. 특정 열만 꺼낸다: `df$score`
5. 조건 필터링: `df[df$score >= 85, ]` (85점 이상 행만 추출)
6. 평균을 구한다: `mean(df$score)`

#### 실습 D. 시각화

1. 히스토그램: `hist(scores)` 실행 후 우하단 **Plots** 탭에서 그래프 확인.
2. 상자그림: `boxplot(scores)`
3. 산점도(두 변수 관계): `plot(df$score)`
4. 막대그래프: `barplot(df$score, names.arg = df$name)`
5. Plots 탭의 **Export** 버튼으로 그래프를 이미지로 저장해 `projects/` 폴더에 보관한다.

#### 실습 E. 실제 공공데이터로 분석 (응용)

1. 공공데이터포털(https://www.data.go.kr) 또는 KOSIS 국가통계포털(https://kosis.kr)에서 관심 있는 통계를 **CSV(파일데이터)**로 내려받는다(로그인 없이 가능).
2. RStudio 메뉴 `Session > Set Working Directory > Choose Directory` 로 CSV가 있는 폴더를 작업 경로로 지정한다.
3. `data <- read.csv("파일이름.csv", fileEncoding = "EUC-KR")` 로 읽어 들인다(한글 깨지면 `fileEncoding`을 `"UTF-8"` 로 바꿔 재시도).
4. `str(data)`, `summary(data)` 로 구조와 분포를 파악한다.
5. 숫자 열 하나를 골라 `hist()` 또는 `boxplot()` 으로 시각화하고, `mean()`·`sd()` 로 요약한다.

## 9. 파이썬을 활용한 자료분석

### 9.1 개요: 왜 배우나

엑셀은 수천 행 수준의 표를 다루기엔 편하지만, 수십만 행의 통계 원자료(KOSIS 마이크로데이터 등)나 반복 작업, 재현 가능한 분석에는 한계가 있다. 파이썬과 pandas는 다음을 가능하게 한다.

- **대용량 처리**: 엑셀이 버거워하는 수십만~수백만 행도 코드 몇 줄로 불러오고 가공한다.
- **재현성**: 같은 코드를 다시 돌리면 항상 같은 결과가 나온다. 분석 과정이 코드로 기록되어 과제 제출과 복습에 유리하다.
- **자동화**: 매달 갱신되는 통계도 데이터만 바꿔 코드를 재실행하면 끝난다.
- **무료·개방 생태계**: pandas(데이터 가공), matplotlib(시각화)는 모두 오픈소스이며, R/엑셀로 배운 개념(평균·집계·피벗·그래프)을 그대로 코드로 옮길 수 있다.

이 과목은 1학년 1학기 입문 과목이므로, 목표는 "파이썬으로 실제 통계 데이터를 불러와 요약하고 그래프 한 장을 그려본다"까지다.

### 9.2 핵심 개념·기능

- **실행 환경 두 가지**
    - **Anaconda**: 내 PC에 파이썬 + pandas + matplotlib + Jupyter Notebook을 한 번에 설치하는 배포판. 오프라인 작업과 로컬 파일 다루기에 좋다.
    - **Google Colab**: 설치 없이 웹 브라우저(구글 계정)에서 바로 코드를 실행하는 클라우드 노트북. 라이브러리가 이미 깔려 있어 입문자에게 가장 빠르다.
- **pandas 핵심 자료구조**
    - **Series**: 1차원 데이터(엑셀의 한 열).
    - **DataFrame**: 2차원 표(엑셀 시트). 행(index)과 열(column)로 구성.
- **데이터 입출력**: `read_csv()`, `read_excel()`로 불러오고 `to_csv()`로 저장.
- **탐색·요약**: `head()`, `tail()`, `info()`, `describe()`, `shape`, `columns`.
- **선택·필터링**: 열 선택 `df['열이름']`, 조건 필터 `df[df['열'] > 값]`, 위치 기반 `iloc`, 라벨 기반 `loc`.
- **집계**: `groupby()`, `mean()`, `sum()`, `count()`, `sort_values()`, `value_counts()`.
- **결측치 처리**: `isnull()`, `dropna()`, `fillna()`.
- **matplotlib 시각화**: `plot()`(선), `bar()`(막대), `hist()`(히스토그램), `scatter()`(산점도)와 제목·축 라벨(`title`, `xlabel`, `ylabel`). 한글 폰트 깨짐 방지 설정 필요.

#### 실습 C: 작은 데이터프레임 직접 만들고 다루기

1. 새 셀에 직접 표를 만든다.
    ```python
    import pandas as pd
    data = {
        "지역": ["서울", "부산", "대구", "인천", "광주"],
        "인구": [940, 330, 240, 295, 145],   # 단위: 만 명(예시)
    }
    df = pd.DataFrame(data)
    df
    ```
2. 구조를 살펴본다.
    ```python
    df.head()        # 앞부분 보기
    df.info()        # 자료형·결측치 확인
    df.describe()    # 숫자 열 요약 통계
    ```
3. 조건으로 걸러보고 정렬한다.
    ```python
    df[df["인구"] >= 300]              # 인구 300만 이상
    df.sort_values("인구", ascending=False)   # 인구 많은 순
    ```

#### 실습 D: 실제 공공 통계(KOSIS) CSV 불러와 분석하기

1. `https://kosis.kr` 에 접속해 관심 통계(예: 시도별 인구)를 검색한다.
2. 통계표 조회 화면에서 `다운로드` 를 눌러 **파일형태를 CSV**로 선택하고 내려받는다.
3. Colab 왼쪽의 폴더 아이콘 > `업로드` 로 받은 CSV를 노트북에 올린다(Anaconda면 노트북과 같은 폴더에 둔다).
4. 파일을 불러온다. 한글 CSV는 인코딩 때문에 깨질 수 있으니 옵션을 시도한다.
    ```python
    df = pd.read_csv("파일이름.csv", encoding="cp949")   # 깨지면 encoding="utf-8" 로 변경
    df.head()
    ```
5. 열 이름과 크기를 확인하고 필요한 열만 추린다.
    ```python
    df.columns      # 열 이름 확인
    df.shape        # (행 수, 열 수)
    ```
6. 집계해본다(열 이름은 실제 파일에 맞게 수정).
    ```python
    df.groupby("시도")["인구수"].sum().sort_values(ascending=False)
    ```

2. 막대그래프를 그린다.
    ```python
    plt.bar(df["지역"], df["인구"])
    plt.title("지역별 인구")
    plt.xlabel("지역")
    ```

4. 완성한 노트북은 `파일 > 다운로드 > .ipynb` 로 저장해 과제로 제출한다.

## 부록. 용어 사전

교재와 본문에 등장하는 생소한 용어를 8개 분류로 정리한 부록이다. 인터랙티브 버전(검색, 분류 필터, 암기 모드, 이상치 실험실 위젯)은 학습 페이지(`docs/data.html`)의 "부록. 데이터 용어 사전"에서 쓸 수 있다.

### 중심경향 (대표값)

| 용어 | 영문 | 뜻 | 도구 표기 |
|------|------|----|----------|
| 중심경향 | central tendency | 데이터가 어디쯤 모여 있는지 요약하는 대표값. 평균/중앙값/최빈값 | - |
| 평균 | mean | 전부 더해 개수로 나눈 값. 이상치에 민감 (60,70,80,90,100 → 80 / 마지막이 1000이면 260) | `AVERAGE` / `mean()` / `.mean()` |
| 중앙값 | median | 정렬했을 때 한가운데 값. 이상치에 강건 (60,70,80,90,1000 → 여전히 80) | `MEDIAN` / `median()` / `.median()` |
| 최빈값 | mode | 가장 자주 나오는 값. 명목척도에서 쓸 수 있는 유일한 대표값 | `MODE.SNGL`(숫자 전용) / `table()` / `.mode()` |
| 강건성 | robustness | 이상치가 끼어도 크게 흔들리지 않는 성질. 중앙값/IQR이 대표적 | - |

### 산포 (퍼짐)

| 용어 | 영문 | 뜻 | 도구 표기 |
|------|------|----|----------|
| 산포 | dispersion | 데이터가 얼마나 퍼져 있는지. A반(78~82) vs B반(40~120)은 평균이 같아도 산포가 다르다 | - |
| 범위 | range | 최댓값 - 최솟값. 단순하지만 이상치 하나에 확 변한다 | `MAX-MIN` / R `max() - min()` (`range()`는 최소/최대 쌍 반환) |
| 편차 | deviation | 각 값 - 평균. 그대로 더하면 항상 0이라 제곱해서 쓴다 | - |
| 분산 | variance | 편차 제곱의 평균. 단위가 제곱이라 직관적으로 읽기 어렵다 | `VAR.S`/`VAR.P` / `var()` / `.var()` |
| 표준편차 | standard deviation | 분산에 루트. 원래 단위로 복원해 "평균에서 대략 이만큼 퍼져 있다" | `STDEV.S`/`STDEV.P` / `sd()` / `.std()` |
| 사분위수 | quartile | 정렬 데이터를 4등분하는 지점. Q1(25%), Q2(50%=중앙값), Q3(75%) | `QUARTILE.INC` / `quantile()` / `.quantile()` |
| IQR(사분위범위) | interquartile range | Q3 - Q1. 가운데 50%의 퍼짐이라 이상치에 강건. 이상치 규칙: Q1-1.5×IQR ~ Q3+1.5×IQR 밖 | - |
| n-1 보정(자유도) | degrees of freedom | 표본의 분산/표준편차는 n-1로 나눠 과소측정을 보정. 도구 간 값 차이의 단골 원인 | `STDEV.S`(n-1) vs `STDEV.P`(n), pandas `ddof=1` |
| Z-점수 | z-score | (값-평균)/표준편차. 평균에서 표준편차 몇 배 떨어졌는지. \|z\|>3 안팎이면 이상치 의심 | - |

### 분포·시각화

| 용어 | 영문 | 뜻 | 도구 표기 |
|------|------|----|----------|
| 분포 | distribution | 값들이 어디에 얼마나 몰려 있는지의 전체 모양 (중심+산포+모양) | - |
| 빈도(도수) | frequency | 각 값/범주가 몇 번 나왔는지. 범주형 요약의 기본 | `COUNTIF` / `table()` / `.value_counts()` |
| 히스토그램 | histogram | 연속형 분포를 구간(bin)별 빈도 막대로. 막대 사이가 붙어 있다 | `hist()` / `.plot(kind="hist")` |
| 계급(빈) | bin | 히스토그램의 구간. 너비에 따라 분포 인상이 달라진다 | 엑셀 축 서식의 계급 구간 너비 |
| 상자그림 | box plot | 5수 요약 기반 그래프. 수염은 1.5×IQR 안 끝 값까지, 그 밖은 이상치 점 | `boxplot()` / `.boxplot()` |
| 시계열 그래프 | time series | 시간 순서에 따른 추세. 꺾은선 기본, 교재는 분산형(X,Y)도 권장 | - |
| 산점도 | scatter plot | 두 수치형 변수의 관계를 점으로 (우상향/우하향/무관) | 엑셀 분산형 / `plot()` / `.plot.scatter(x=, y=)` |
| 왜도(치우침) | skewness | 꼬리가 늘어진 방향. 오른쪽 꼬리가 길면 평균 > 중앙값 | - |
| 막대 차트 | bar chart | 범주 비교용(막대 간격 있음). 연속형 분포는 히스토그램(간격 없음) | - |

### 척도·데이터

| 용어 | 영문 | 뜻 |
|------|------|----|
| 변수 | variable | 측정 항목 하나(키, 성별). 표의 열 하나 |
| 관측치 | observation | 한 개체의 측정값 묶음(한 사람). 표의 행 하나 |
| 범주형 변수 | categorical | 분류/이름(성별, 시도명). 빈도/비율로 요약 |
| 수치형(양적) 변수 | numerical | 수량(키, 소득). 평균/표준편차로 요약. 이산형/연속형으로 세분 |
| 명목척도 | nominal | 순서 없는 이름표(혈액형, 우편번호). 최빈값만 의미 |
| 순서척도 | ordinal | 순서만 있고 간격 불일정(만족도, 학점). 중앙값까지 의미 |
| 등간척도 | interval | 간격 일정, 절대 0 없음(섭씨, 연도). 배수 비교 무의미 |
| 비율척도 | ratio | 절대 0 있음(키, 소득). 모든 사칙연산 의미. 정보량: 명목<순서<등간<비율 |

### 표본·추론

| 용어 | 영문 | 뜻 |
|------|------|----|
| 모집단 | population | 알고 싶은 대상 전체 |
| 표본 | sample | 모집단에서 뽑은 일부. 현실 분석의 대부분 |
| 모수 | parameter | 모집단의 특성치(모평균 등). 보통 직접 알 수 없다 |
| 통계량 | statistic | 표본에서 계산한 특성치(표본평균 등). 모수 추정에 사용 |
| 기술통계 | descriptive statistics | 가진 데이터를 요약/묘사. 이 과목의 중심 |
| 추론통계 | inferential statistics | 표본으로 모집단을 추정/검정. 후속 과목에서 본격화 |
| 편향 | bias | 표본이 모집단을 대표하지 못해 결과가 치우침 (도서관 이용자만 조사하는 식) |

### 관계·회귀

| 용어 | 영문 | 뜻 | 도구 표기 |
|------|------|----|----------|
| 상관계수 | correlation (r) | 두 수치형 변수의 직선 관계를 -1 ~ +1로 요약 | `CORREL` / `cor()` / `.corr()` |
| 상관 ≠ 인과 | correlation vs causation | 같이 움직여도 원인-결과는 아니다 (아이스크림과 익사: 원인은 더위) | - |
| 단순선형회귀 | simple linear regression | y = a + bx. 기울기 b = x가 1 늘 때 y의 평균 변화량, 절편 a = x=0일 때 예측값 | `SLOPE`/`INTERCEPT` / `lm(y ~ x)` |
| R제곱(결정계수) | R-squared | 회귀직선이 y 흩어짐의 몇 %를 설명하는지(0~1). 단순회귀에선 r의 제곱 | `RSQ` / `summary(fit)` |
| 추세선 | trendline | 산점도에 가장 잘 맞는 직선/곡선. 엑셀 우클릭으로 회귀 체험 | 추세선 옵션의 수식/R제곱 표시 |

| 용어 | 뜻 | 도구 표기 |
