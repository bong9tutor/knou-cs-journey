# 컴퓨터의이해 학습 노트

> 한국방송통신대학교 컴퓨터과학과 1학년 1학기 과목 "컴퓨터의이해"의 학습 정리 노트다. 컴퓨터를 처음 접하는 사람도 정보사회의 기본 소양을 갖추도록 설계된 입문/교양 성격의 과목으로, 코딩 실습 없이 개념과 원리를 폭넓게 다룬다.
> 이 노트는 KNOU 공식 교재 목차(15장)에 1:1로 대응하도록 구성했다. 교재 서지와 장 구성은 출판문화원 공식 페이지로 교차 검증했으며, 평가 방식은 방송대 전체 공통 학칙 기준이라 학기별 강의계획서에서 최종 확인이 필요한 항목은 "확인 필요"로 표기했다.

## 과목 개요

- **과목 성격**: 이론/교양 기초 (프로그래밍 실습 없음, `projects/` 폴더 없는 이론 과목)
- **대상**: 1학년 1학기, 컴퓨터 입문자
- **주의**: "컴퓨터의이해"는 "컴퓨터과학개론"과 **다른 과목**이다. 컴퓨터과학개론이 전공 심화의 출발점(자료 표현, 알고리즘, 운영체제, 네트워크를 학문적으로 다룸)이라면, 컴퓨터의이해는 정보사회의 일원으로서 갖춰야 할 컴퓨터 기초 소양과 시사 흐름(4차 산업혁명, 유비쿼터스, 모바일 서비스 등)을 폭넓고 쉽게 소개하는 데 초점이 있다.

## 1. 정보사회와 컴퓨터

정보가 핵심 자원이 된 사회와 그 중심에 있는 컴퓨터의 기본 개념을 잡는 단원이다.

### 정보사회와 정보혁명
- **정보사회(information society)**: 정보의 생산, 처리, 유통이 경제와 생활의 중심이 된 사회. 농업혁명 → 산업혁명 → 정보혁명으로 이어지는 흐름의 결과다.
- **정보혁명**: 컴퓨터와 통신기술의 발전으로 정보의 양과 속도가 폭발적으로 늘어난 변화. 과거에는 토지/자본/노동이 핵심 자원이었다면, 정보사회에서는 "정보와 지식"이 가장 중요한 자원이 된다.

### 데이터와 정보의 차이 (핵심 구분)

| 구분 | 데이터(Data) | 정보(Information) |
| --- | --- | --- |
| 의미 | 가공 전의 원시 사실/값 | 데이터를 가공해 의미를 부여한 결과 |
| 예시 | "23, 25, 21" (기온 숫자) | "이번 주 평균 기온 23도, 어제보다 따뜻함" |
| 관계 | 입력 | 처리 결과(출력) |

- 더 나아가 데이터 → 정보 → **지식(knowledge)** → **지혜(wisdom)** 로 가공 수준이 올라간다는 DIKW 위계를 함께 기억하면 좋다.

### 컴퓨터 시스템의 구성과 기능
- 컴퓨터의 5대 기능: **입력 - 기억(저장) - 연산 - 제어 - 출력**.
- 하드웨어는 크게 중앙처리장치(CPU), 기억장치(주기억/보조기억), 입출력장치로 나뉜다.
- 컴퓨터의 기본 동작 사이클: 입력 → 처리(연산/제어) → 출력, 그리고 필요한 데이터는 기억장치에 저장.

### 컴퓨터의 종류
- 처리 규모/용도에 따라: 슈퍼컴퓨터, 메인프레임, 미니컴퓨터(중형), 마이크로컴퓨터(PC), 그리고 스마트폰/임베디드 기기 등.
- 데이터 표현 방식에 따라: 디지털 컴퓨터(대부분), 아날로그 컴퓨터, 하이브리드 컴퓨터.

### 유비쿼터스 시대의 도래
- 언제 어디서나 컴퓨터와 네트워크를 사용할 수 있는 환경의 등장. (자세한 내용은 14장에서 다룸)

## 2. 컴퓨터와 통신산업의 발전

계산도구의 역사부터 현대 컴퓨터 산업, 그리고 컴퓨터와 통신의 결합까지의 흐름을 다룬다.

### 계산도구의 역사
- **초창기 계산도구**: 주판 → 파스칼의 기계식 계산기(덧셈/뺄셈) → 라이프니츠 계산기(곱셈/나눗셈) → 배비지의 해석기관(현대 컴퓨터의 개념적 원형).
- **전자식 계산기의 등장**: ENIAC(진공관 기반 최초의 대규모 전자식 컴퓨터)이 대표적. 다만 ENIAC은 배선을 바꿔 프로그램을 했다.

### 프로그램 내장방식 (가장 중요한 개념)
- **폰 노이만(von Neumann) 구조 / 프로그램 내장방식(stored-program)**: 프로그램(명령어)을 데이터처럼 기억장치에 저장해 두고, CPU가 순차적으로 꺼내 실행하는 방식. 오늘날 거의 모든 컴퓨터의 기본 구조다.
- 의의: 배선 교체 없이 소프트웨어만 바꾸면 다른 작업을 수행 -> 범용 컴퓨터의 탄생.

### 컴퓨터 산업의 발전 단계 (세대 구분)

| 세대 | 핵심 소자 | 특징 |
| --- | --- | --- |
| 1세대 | 진공관 | 크고 발열 많음, ENIAC |
| 2세대 | 트랜지스터 | 소형화, 신뢰성 향상 |
| 3세대 | 집적회로(IC) | 더 작고 빨라짐 |
| 4세대 | 고밀도 집적회로(LSI/VLSI), 마이크로프로세서 | PC 대중화 |
| 5세대 | 인공지능/병렬처리 지향 | 지능형 컴퓨터 |

### 컴퓨터와 통신의 결합
- 독립적으로 발전하던 컴퓨터 기술과 통신 기술이 만나 네트워크/인터넷 시대를 열었다. 컴퓨터 산업의 미래는 AI, 클라우드, 모바일과의 융합 방향으로 나아간다.

## 3. 처리장치와 데이터 처리

CPU가 데이터를 어떻게 저장하고 표현하며 명령어를 실행하는지 다룬다.

### 처리장치(CPU)의 구성
- **제어장치(Control Unit)**: 명령어를 해석하고 각 장치에 제어 신호를 보냄.
- **연산장치(ALU, 산술논리연산장치)**: 사칙연산과 논리연산(AND/OR/NOT, 비교 등)을 수행.
- **레지스터(Register)**: CPU 내부의 초고속 임시 저장 공간.

### 데이터의 저장과 표현 (2진수)
- 컴퓨터는 모든 데이터를 **0과 1(2진수)** 로 표현한다. 전기 신호의 on/off에 대응하기 때문.
- **비트(bit)**: 0 또는 1, 정보의 최소 단위. **바이트(byte)**: 8비트, 문자 1개 표현 단위.
- 단위 위계: bit < byte < KB < MB < GB < TB.
- 문자 표현: 아스키(ASCII, 영문 중심), 유니코드(Unicode, 한글 포함 전 세계 문자).
- 음수/소수 표현: 정수는 2의 보수, 실수는 부동소수점(floating point) 방식.

### 컴퓨터 명령어의 실행 (명령어 사이클)
- 한 개의 명령어는 **인출(Fetch) → 해석(Decode) → 실행(Execute)** 과정을 거친다.
- 주기억장치는 주소(address)로 각 저장 위치를 구분하고, CPU는 그 주소를 통해 데이터/명령어를 읽고 쓴다.

### 기억장치의 발달과 반도체기억장치
- **RAM(주기억, 휘발성)**: 전원이 꺼지면 내용이 사라짐. 실행 중인 프로그램/데이터를 임시 저장.
- **ROM(비휘발성)**: 전원이 꺼져도 유지. 부팅에 필요한 기본 프로그램 등 저장.

## 4. 컴퓨터의 입출력

사람과 컴퓨터, 컴퓨터와 컴퓨터가 데이터를 주고받는 방법을 다룬다.

### 입력장치 (사람 → 컴퓨터)
- 키보드, 마우스, 터치스크린, 스캐너, 마이크, 카메라, 바코드/QR 리더, 센서 등.

### 출력장치 (컴퓨터 → 사람)
- 모니터(디스플레이), 프린터, 스피커, 프로젝터 등.

### 입력 데이터 편집과 출력 정보
- 입력된 원시 데이터는 오류 검사, 형식 변환 등의 **편집** 과정을 거쳐 처리된다.
- 출력은 화면 출력(soft copy)과 인쇄 출력(hard copy)으로 나뉜다.

### 입출력의 미래
- 음성 인식, 제스처 인식, VR/AR 인터페이스, 뇌-컴퓨터 인터페이스(BCI) 등 인간 친화적 입출력으로 발전.

> 참고: 이 장(컴퓨터의 입출력)은 중간과제물 출제 이력이 있는 단원이므로 입출력장치 분류와 흐름을 정확히 정리해 둘 것.

## 5. 보조기억장치와 데이터 저장

주기억장치를 보완하는 대용량/비휘발성 저장장치를 다룬다.

### 주기억장치 vs 보조기억장치

| 구분 | 주기억장치(RAM) | 보조기억장치(HDD/SSD 등) |
| --- | --- | --- |
| 속도 | 빠름 | 느림(상대적) |
| 용량 | 작음 | 큼 |
| 휘발성 | 휘발성(전원 끄면 소멸) | 비휘발성(영구 보관) |
| 용도 | 실행 중 임시 저장 | 장기 보관 |

### 저장장치의 종류와 진화
- **자기테이프**: 순차 접근(앞에서부터 차례로 읽음), 백업용.
- **자기디스크(HDD)**: 회전하는 자기 원판에 데이터 저장, 임의 접근(random access) 가능.
- **광디스크**: CD, DVD, 블루레이. 레이저로 읽고 씀.
- **반도체 기억장치(SSD, USB 메모리, SD카드)**: 플래시 메모리 기반, 기계적 움직임 없어 빠르고 충격에 강함. 현재 HDD를 빠르게 대체 중.

### 버퍼와 캐시
- **버퍼(buffer)**: 속도 차이가 나는 두 장치 사이에서 데이터를 임시로 모아 두는 공간.
- **캐시(cache)**: 자주 쓰는 데이터를 빠른 곳에 미리 두어 접근 속도를 높이는 기법.

### 클라우드 기반 저장
- 데이터를 내 기기가 아닌 인터넷상의 서버에 저장하고 공유(예: 구글 드라이브, OneDrive, iCloud). 어디서나 접근 가능하지만 보안/네트워크 의존이라는 양면성이 있다.

## 6. 소프트웨어와 정보 시스템

하드웨어를 움직이는 소프트웨어와, 조직에서 활용되는 정보 시스템을 다룬다.

### 소프트웨어의 종류

| 구분 | 설명 | 예시 |
| --- | --- | --- |
| 시스템 소프트웨어 | 하드웨어를 관리/제어하고 응용 SW가 동작할 기반 제공 | 운영체제(Windows, macOS, Linux), 컴파일러, 유틸리티 |
| 응용 소프트웨어 | 특정 업무/목적을 위한 SW | 워드, 엑셀, 웹브라우저, 게임 |

- **운영체제(OS)**: 가장 핵심적인 시스템 소프트웨어. 자원 관리(CPU/메모리/입출력), 사용자 인터페이스 제공.

### 소프트웨어의 개발
- 개발 단계(소프트웨어 생명주기): 요구분석 → 설계 → 구현(코딩) → 테스트 → 유지보수.

### 소프트웨어의 사용권(라이선스)
- **상용 소프트웨어**: 돈을 내고 사용권 구매.
- **셰어웨어**: 일정 기간/기능 무료 체험 후 결제.
- **프리웨어**: 무료 사용, 저작권은 개발자에게.
- **오픈소스(open source)**: 소스코드 공개, 자유로운 사용/수정/배포 (예: 리눅스).

### 기업활동과 정보 시스템
- **MIS(경영정보시스템)**, **ERP(전사적 자원관리)**, **DSS(의사결정지원시스템)** 등 조직의 의사결정과 업무를 돕는 정보 시스템으로 발전해 왔다.

## 7. 데이터베이스

대량의 데이터를 체계적으로 저장/관리하는 방법을 다룬다.

### 데이터베이스의 개념
- **데이터베이스(DB)**: 여러 사용자가 공유할 수 있도록 통합/저장해 둔 데이터의 집합. 중복을 줄이고 일관성을 유지하는 것이 목표.
- 파일 시스템 방식의 문제(데이터 중복, 불일치, 의존성)를 해결하기 위해 등장.

### 데이터베이스 관리 시스템(DBMS)
- **DBMS**: 데이터베이스를 만들고 관리하며, 사용자가 데이터를 쉽게 저장/검색/수정/삭제하도록 돕는 소프트웨어 (예: Oracle, MySQL, MS SQL Server).
- 사용자는 **질의어(SQL)** 로 데이터베이스에 명령을 내린다.

### 3단계 스키마 구조

| 단계 | 의미 |
| --- | --- |
| 외부 스키마 | 사용자/응용 프로그램이 보는 부분적 관점(뷰) |
| 개념 스키마 | 전체 데이터베이스의 논리적 구조(조직 전체 관점) |
| 내부 스키마 | 실제 저장 방식(물리적 구조) |

- 이 구조 덕분에 **데이터 독립성**(저장 방식이 바뀌어도 응용 프로그램 영향 최소화)이 확보된다.

### 데이터베이스의 장단점
- 장점: 중복 최소화, 데이터 공유, 일관성/무결성 유지, 보안.
- 단점: 시스템 구축/운영 비용, 복잡성, 장애 시 영향 큼.

### 데이터 웨어하우스
- 의사결정 지원을 위해 여러 곳의 데이터를 모아 분석용으로 저장하는 대규모 저장소. (빅데이터 분석의 기반)

## 8. 정보통신기술과 우리 사회

정보통신기술(ICT)이 사회 각 분야를 어떻게 바꾸는지 다룬다.

### 정보통신기술(ICT)이란
- Information and Communication Technology. 정보를 생산/처리/전달하는 기술 전반.

### 분야별 정보통신기술 활용

| 분야 | 활용 예시 |
| --- | --- |
| 행정/공공 | 전자정부(민원 온라인 처리), 전자투표 |
| 교육 | 이러닝, 원격수업(방송대가 대표 사례) |
| 의료 | 원격진료, 전자의무기록(EMR) |
| 금융 | 인터넷/모바일 뱅킹, 핀테크, 간편결제 |
| 상거래 | 전자상거래(e-commerce), 온라인 쇼핑 |
| 가정 | 스마트홈, IoT 가전 |

- 동시에 **정보격차(디지털 디바이드)**, 사생활 침해, 정보 과부하 같은 사회적 그림자도 함께 이해해야 한다.

## 9. 4차 산업혁명과 미래사회

현재 진행 중인 기술 혁명과 핵심 기술을 다룬다.

### 산업혁명의 단계

| 구분 | 핵심 동력 |
| --- | --- |
| 1차 | 증기기관(기계화) |
| 2차 | 전기(대량생산) |
| 3차 | 컴퓨터/인터넷(정보화, 자동화) |
| 4차 | AI, IoT, 빅데이터, 초연결/초지능(융합) |

### 4차 산업혁명 핵심 기술
- **인공지능(AI)**: 학습/추론/판단을 하는 시스템. 머신러닝, 딥러닝 포함.
- **사물인터넷(IoT)**: 사물에 센서/통신 기능을 부여해 인터넷에 연결. (예: 스마트 냉장고, 웨어러블, 스마트 가로등, 커넥티드 카)
- **빅데이터(Big Data)**: 방대하고 다양한 데이터를 빠르게 수집/분석해 가치를 뽑아냄. 3V(Volume, Variety, Velocity).
- **클라우드 컴퓨팅**: 컴퓨팅 자원을 인터넷으로 빌려 씀.
- 그 외: 로봇, 자율주행, 3D 프린팅, 블록체인, AR/VR.

> 참고: 이 장(사물인터넷 사례)은 중간과제물 출제 이력이 있다. IoT 적용 사례를 분야별로 4개 이상 구체적으로 정리해 둘 것.

## 10. 정보통신

데이터가 멀리 떨어진 곳까지 전달되는 원리와 네트워크 구성을 다룬다.

### 정보통신의 기본 요소
- **송신자 - 전송매체 - 수신자**, 그리고 약속(프로토콜)이 필요하다.
- 데이터를 전기/광 신호로 바꿔 전송하고, 받는 쪽에서 다시 복원한다.

### 정보통신 방식

| 분류 기준 | 종류 |
| --- | --- |
| 전송 방향 | 단방향(simplex), 반이중(half-duplex), 전이중(full-duplex) |
| 신호 형태 | 아날로그 전송, 디지털 전송 |
| 전송 단위 | 직렬 전송, 병렬 전송 |

### 정보통신망(네트워크)의 유형

| 유형 | 범위 | 예 |
| --- | --- | --- |
| LAN | 근거리(건물/사무실) | 회사 내부망 |
| MAN | 도시 규모 | 도시 통신망 |
| WAN | 광역(국가/대륙) | 인터넷 백본 |

- 토폴로지(망 구성 형태): 성형(star), 버스형(bus), 링형(ring), 망형(mesh).

### 통신 프로토콜
- **프로토콜(protocol)**: 통신을 위한 약속(규칙). 대표적으로 인터넷의 **TCP/IP**. 계층 모델(OSI 7계층 / TCP/IP 4계층)로 역할을 나눈다.

## 11. 인터넷

전 세계 컴퓨터를 연결하는 거대한 네트워크인 인터넷을 다룬다.

### 인터넷의 개요
- 수많은 네트워크가 TCP/IP 프로토콜로 연결된 "네트워크의 네트워크". 특정 주인이 없는 개방형 구조.

### 인터넷 주소

| 종류 | 설명 | 예 |
| --- | --- | --- |
| IP 주소 | 인터넷에 연결된 각 기기의 숫자 주소 | 203.0.113.1 (IPv4), IPv6는 더 길고 큰 주소공간 |
| 도메인 이름 | 사람이 외우기 쉬운 문자 주소 | www.knou.ac.kr |
| DNS | 도메인 이름 ↔ IP 주소 변환 시스템 | 전화번호부 역할 |
| URL | 인터넷 자원의 위치 주소 | https://www.knou.ac.kr/page |

### 인터넷 서비스
- **WWW(웹)**: HTTP 프로토콜, 브라우저로 하이퍼텍스트 문서 열람.
- **이메일(SMTP/POP3/IMAP)**, **FTP(파일 전송)**, **클라우드/스트리밍/SNS** 등.

### 인터넷 응용
- 전자상거래, 검색엔진, 소셜미디어, IoT, 웹 기반 서비스 전반.

## 12. 정보보안

연결된 사회에서 데이터와 시스템을 지키는 방법을 다룬다.

### 정보보안의 3대 목표 (CIA)

| 목표 | 의미 |
| --- | --- |
| 기밀성(Confidentiality) | 인가된 사람만 정보에 접근 |
| 무결성(Integrity) | 정보가 임의로 변조되지 않음 |
| 가용성(Availability) | 필요할 때 정보/서비스를 사용 가능 |

### 위협 요소
- **해킹**: 시스템에 무단 침입해 정보를 빼내거나 파괴.
- **악성 프로그램(malware)**: 바이러스(다른 파일에 기생), 웜(스스로 복제/전파), 트로이목마(정상 프로그램 위장), 랜섬웨어(파일 암호화 후 금전 요구), 스파이웨어.
- 사회공학적 공격: 피싱(가짜 사이트/메일로 정보 탈취), 파밍, 스미싱(문자 이용).

### 보안 강화 방법
- **암호화(encryption)**: 데이터를 알아볼 수 없게 변환. 대칭키/공개키 방식.
- **인증**: 비밀번호, 생체인증, OTP, 2단계 인증(2FA).
- **방화벽**, 백신, 보안 패치 최신화, 백업.

## 13. 멀티미디어

문자, 소리, 이미지, 영상 등 다양한 미디어를 통합 처리하는 기술을 다룬다.

### 멀티미디어란
- 텍스트, 이미지, 사운드, 애니메이션, 동영상 등 두 가지 이상의 미디어를 디지털로 통합해 다루는 것. 특징은 **디지털화, 통합성, 상호작용성(interactivity)**.

### 디지털 미디어의 표현

| 미디어 | 핵심 개념 | 대표 형식 |
| --- | --- | --- |
| 이미지 | 비트맵(픽셀 기반) vs 벡터(수식 기반), 해상도, 색 깊이 | JPEG, PNG, GIF, BMP |
| 사운드 | 샘플링(아날로그→디지털), 샘플링 비율 | MP3, WAV |
| 동영상 | 프레임의 연속, 압축(코덱) | MP4, AVI |

- **압축**: 데이터 크기를 줄이는 기술. 무손실(원본 완전 복원) vs 손실(일부 정보 버려 더 작게).
- **아날로그 → 디지털 변환**: 표본화(sampling) → 양자화(quantization) → 부호화(encoding).

### 멀티미디어의 활용
- 교육(이러닝), 게임, 영화/방송, 가상현실(VR)/증강현실(AR), 디지털 광고 등.

## 14. 유비쿼터스

언제 어디서나 컴퓨팅이 가능한 환경을 다룬다.

### 유비쿼터스의 개념
- **유비쿼터스(ubiquitous)**: 라틴어로 "어디에나 존재한다"는 뜻. 사용자가 의식하지 않아도 일상 곳곳에 컴퓨터가 스며들어 언제 어디서나 정보를 활용하는 환경.
- 특성: 보이지 않음(invisible), 어디에나 있음, 자연스러운 상호작용, 네트워크 연결.

### 관련 기술과 사물인터넷(IoT)
- **RFID/NFC**(근거리 무선 인식), 센서, 무선 네트워크가 기반.
- **사물인터넷(IoT)**: 사물들이 서로 연결되어 데이터를 주고받음. 유비쿼터스 비전을 실현하는 핵심.

### 스마트시티
- IoT, 빅데이터, AI를 도시 인프라(교통, 에너지, 안전, 환경)에 적용해 효율적이고 지속가능하게 운영하는 도시. (예: 스마트 교통 신호, 스마트 미터, 스마트 주차)

### 연구 동향과 전망
- 헬스케어, 스마트홈, 웨어러블, 자율주행 등으로 확장 중.

## 15. 모바일 서비스

스마트폰을 중심으로 한 모바일 환경과 서비스를 다룬다.

### 모바일 서비스의 개요
- 이동 중에도 무선 네트워크를 통해 정보/서비스를 이용하는 것. 스마트폰 보급으로 폭발적으로 성장.
- 이동통신 세대: 1G(아날로그 음성) → 2G(디지털, 문자) → 3G(영상통화, 데이터) → 4G(LTE, 고속 데이터) → 5G(초고속/초저지연/초연결).

### 모바일 운영체제

| OS | 특징 |
| --- | --- |
| 안드로이드(Android) | 구글, 오픈소스 기반, 다양한 제조사 채택, 점유율 높음 |
| iOS | 애플, 폐쇄적/통합적, 자사 기기 전용 |

### 모바일 서비스 사례
- 모바일 앱 마켓(앱스토어, 플레이스토어), 모바일 결제/뱅킹, 위치기반 서비스(LBS), 모바일 메신저/SNS, O2O 서비스(배달/택시).

### 웨어러블 기기
- 몸에 착용하는 컴퓨팅 기기: 스마트워치, 스마트밴드(건강 측정), 스마트글래스, 무선 이어버드 등. 헬스케어/IoT와 결합해 성장 중.

## 시험·과제 대비 핵심 정리

- **자주 나오는 개념 구분**: 데이터 vs 정보 / 주기억 vs 보조기억(휘발성) / 시스템 SW vs 응용 SW / LAN vs WAN / 비트맵 vs 벡터 / 안드로이드 vs iOS.
- **반드시 외울 구조**: 프로그램 내장방식(폰 노이만), 컴퓨터의 5대 기능, 명령어 사이클(인출-해석-실행), DB 3단계 스키마, 정보보안 CIA, 산업혁명 1~4차, 이동통신 1G~5G.
