# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 이 레포의 성격

방송통신대학교 컴퓨터과학과 4년 과정의 **학습 기록 저장소**다. 일반적인 소프트웨어 프로젝트가 아니라 강의 노트, 과제, 실습 결과물을 학년·학기·과목별로 모아두는 콘텐츠 레포다.

따라서 레포 최상위에는 빌드·테스트·린트 시스템이 없다. 빌드/실행 도구는 각 프로그래밍 과목의 `projects/` 안에서 그 과목 언어에 맞게(예: Python `venv`, Java `target/`, Node `node_modules/`) 개별적으로 추가된다. 새 도구 체인을 도입할 때는 해당 `projects/` 폴더 안에서 독립적으로 구성하고, 레포 전역에 영향을 주는 설정은 만들지 않는다.

현재 상태: 디렉토리 골격만 있고 실제 콘텐츠는 비어 있다. 빈 폴더는 `.gitkeep`으로 구조를 유지한다 (총 117개). 콘텐츠가 채워지면 해당 폴더의 `.gitkeep`은 제거되므로 이 개수는 점차 줄어든다.

수강 과목은 개설된 **전 과목 49과목**(약 145학점)을 정규 8학기에 나눠 듣는 구성이다 (학기당 6과목·18학점, 연 36학점으로 학기당 18학점 제한 내). 방통대 컴퓨터과학과는 전공필수 지정이 없어 졸업학점(총 130: 전공 51↑, 교양 24↑)은 충분히 초과 달성되며, 전공 초과분은 일반선택으로 인정된다. 과목을 빼거나 더할 때는 이 학점 구조와 학기당 18학점 제한을 깨지 않도록 한다.

## 디렉토리 구조와 명명 규칙

```
yearN/N-S_과목명/{notes,assignments,projects}
```

- `yearN/` - 학년 (year1 ~ year4)
- `N-S_과목명` - 과목 폴더. 접두사 `N-S`가 **학년-학기**를 인코딩한다 (예: `1-1_`=1학년 1학기, `3-2_`=3학년 2학기). 과목명은 **공백·쉼표 없이** 붙여 쓴다 (예: `1-1_파이썬프로그래밍기초`, `2-2_오픈소스기반데이터분석`). 정식 과목명에 특수문자가 있으면 제거한다 (예: '성,사랑,사회' → `4-2_성사랑사회`).
- `원격대학교육의이해`는 매 학기 개설되지만 신입생 적응 과목이라 실제 수강은 1회뿐이므로 1학년 1학기(`1-1_원격대학교육의이해`)에만 폴더를 둔다.
- 과목 하위 폴더: `notes/`(강의 노트·요약), `assignments/`(과제·출석수업 TMA), `projects/`(실습·프로젝트 결과물)

### 프로그래밍 과목 vs 이론 과목

`projects/` 폴더 유무가 과목 성격을 구분한다. **프로그래밍 과목만 `projects/`를 가진다.** 과목명에 '프로그래밍'이 들어가도 코딩 실습이 없으면 이론으로 본다 (예: `프로그래밍언어론`, `컴파일러구성`은 개념·서술형 과제 과목이라 이론).

- `projects/` 있음 (프로그래밍/실습): 파이썬프로그래밍기초(Python), 데이터정보처리입문(Excel/R/Python), HTML5웹프로그래밍(HTML/CSS/JS), Java프로그래밍(Java), 데이터베이스시스템(SQL), 알고리즘, 인공지능, 컴퓨터그래픽스, 모바일앱프로그래밍, C프로그래밍(C), 자료구조(C), 오픈소스기반데이터분석(R/Python), JSP프로그래밍(JSP/Java), UNIX시스템(셸 스크립트), 시뮬레이션, 머신러닝(Python), 딥러닝(Python), 클라우드컴퓨팅
- `projects/` 없음 (이론): 유비쿼터스컴퓨팅개론, 디지털논리회로, 운영체제, 소프트웨어공학, 정보통신망, 컴퓨터보안, 그래픽커뮤니케이션, 컴퓨터과학개론, AI리터러시, 프로그래밍언어론, 컴퓨터구조, 컴파일러구성, 빅데이터의이해와활용, 그리고 교양·기타 과목(대학영어, 심리학에게묻다, 원격대학교육의이해, 대중영화의이해, 대학수학의이해, 선형대수, 경제학의이해, 생활법률, 경영전략론, 성사랑사회, 컴퓨터의이해, 세계의정치와경제, 사진의이해, 테마가있는음악여행, 환경과건강, 한국사의이해, 생활과건강)

새 과목 폴더를 만들 때 이 구분을 따른다. 이론 과목에는 `projects/`를 만들지 않는다. `.editorconfig`에 이미 `*.{h,hpp,cpp,inl,cs}` 전용 규칙이 있어, 컴퓨터그래픽스 실습은 C/C++ 또는 C# 기반을 전제로 한다.

### `.gitkeep` 처리

빈 폴더 유지용이다. 어떤 폴더에 실제 파일을 추가하면 그 폴더의 `.gitkeep`은 제거한다 (구조 표식이 더 이상 필요 없으므로).

## 파일 작성 규약

`.editorconfig`가 전 파일에 강제하는 규칙 (Windows 환경 + 한글 깨짐 방지가 핵심):

- 인코딩 **UTF-8**, 줄바꿈 **CRLF**
- 들여쓰기 **스페이스 4칸** (탭 금지)
- 파일 끝 빈 줄 추가, 줄 끝 공백 제거

`.gitignore`는 언어별 빌드 산출물을 이미 처리한다 (Python `__pycache__`/`.venv`, Java `target`/`build`/`out`/`*.jar`, Node `node_modules`/`dist`, C/C++ `*.o`/`*.obj`/`*.exe`/`*.a`/`*.lib`/`*.dll` 등과 CMake 산출물). 단 `**/libs/*.jar`와 C/C++ 라이브러리(`**/libs/*.a`, `*.lib`, `*.so`, `*.dll`)는 추적 유지하므로 라이브러리 파일은 `libs/`에 둔다.

## 텍스트·문서 작성 규칙

- 기본 언어는 **한국어**.
- **em dash(`—`, U+2014) 사용 금지.** 답변·코드 주석·문서 어디에서도 쓰지 않고 ASCII 하이픈(`-`)으로 대체한다. en dash(`–`), ellipsis(`…`), 중간점(`·`), 스마트 따옴표(`""''`) 등 다른 타이포그래피 문자는 이 규칙의 대상이 아니므로 자유롭게 사용 가능하다.
