````text
[3] 각 과목 페이지 본문 - 회차 학습 양식
각 과목 페이지(행)의 본문에는 아래 구조를 넣는다.

(3-1) 학습 단계(Step)는 레포의 과목별 학습 마크다운에서 가져온다 - 중요
이 레포에는 과목별 학습 문서가 있고, Notion의 Step 토글은 이 문서들을 "정본"으로 삼아 채운다. 임의로 지어내지 말고 아래 우선순위로 참고한다.
  1) resources/study-noti/<과목명>/contents.md  - Step 목록의 1차 정본
     - "## Step N: 제목" 형태의 학습 단계 목록이다.
     - 각 "## Step N: 제목"을 그대로 Notion 토글 "Step N · 제목"으로 옮긴다 (콜론 ": "을 가운뎃점 " · "으로만 바꾼다).
     - Step 개수·순서·제목을 임의로 줄이거나 바꾸지 않는다 (예: 파이썬프로그래밍기초는 Step 1~29 전부, C프로그래밍은 1~31 전부).
  2) year<학년>/<학기>_<과목명>/notes/<과목명>_학습노트.md  - 단원 구분과 상세 개념의 출처
     - "## N. 단원명" 구조의 상세 노트다. Step들을 "## 단원명" 헤딩으로 묶을 때 이 단원 구분(또는 contents.md의 주차 표기)을 참고한다.
     - 안내 인용구의 "학습 노트" 링크가 가리키는 대상이기도 하다.
  3) 위 학습 문서가 아직 없는 과목은, 빈 "Step 1" 토글 하나만 만들고 이후 사용자가 채운다.

(3-2) 본문 구조
- 맨 위에 안내 인용구 2줄(각 줄 앞에 "> "). 둘째 줄 괄호 안에는 해당 과목 학습노트의 레포 경로를 넣는다(노트가 없으면 괄호째 생략):
  > 매일 학습한 뒤, 해당 Step 토글 안에 "N회차" 토글을 만들어 그날 학습 내용을 자유롭게 정리하세요. 한 Step은 여러 번 복습하므로 회차를 늘려가며 기록합니다.
  > 상세 개념은 해당 과목의 학습 노트(year1/1-1_파이썬프로그래밍기초/notes/파이썬프로그래밍기초_학습노트.md)를 참조하세요.
- 그 아래 "---" 구분선
- 단원/주차를 "## 단원명" 헤딩으로 묶고, 그 아래 해당 단원에 속한 Step마다 아래 토글을 둔다. (각 Step 토글 안에는 비어 있는 "1회차" 토글 하나만 둔다)

<details>
<summary>Step 1 · (학습 단계 제목)</summary>
	<details>
	<summary>1회차</summary>
	</details>
</details>

토글 주의사항(중요):
- Notion 토글은 HTML <details><summary>제목</summary> 내용 </details> 형태로만 렌더된다. 화살표(쉐브론) 기호나 "-" 들여쓰기 방식으로는 토글이 만들어지지 않고 일반 텍스트로 남는다.
- <summary> 다음 줄에는 빈 줄을 넣지 말고, 탭으로 들여쓴 중첩 "1회차" <details>를 바로 붙인다.
- 처음에는 각 Step 토글 안에 비어 있는 "1회차" 토글 하나만 둔다. 같은 단계를 다시 복습할 때마다 "2회차", "3회차" 토글을 중첩으로 추가해 회차별로 자유롭게 기록한다. 고정 양식(학습일/핵심정리 등)은 두지 않는다.

[4] 보드/뷰 구성
- 기본 표(Table) 뷰 외에 보드(Board) 뷰를 만든다.
  - "학기별" 보드: 학기로 그룹화, 카드 정렬은 중요도 옵션 순서(높음 -> 중간 -> 낮음)대로, 카드에 중요도와 계열을 표시(SHOW "중요도", "계열").
  - "상태판" 보드: 상태(시작 전 / 진행 중 / 완료)로 그룹화, 정렬은 중요도 순, 카드에 중요도와 진도율을 표시(SHOW "중요도", "진도율").
- 각 보드 카드에 중요도 속성을 표시(SHOW)해 과목별 중요도를 한눈에 확인할 수 있게 한다(중요도 select 칩이 옵션 색으로 보임).
- 전체적으로 정렬 기준은 "중요도(전공 핵심도)"를 우선한다. (중요도는 select라 옵션 순서대로 정렬된다)
- 참고: 보드 "카드 색상(Color cards by)"은 Notion UI 전용 설정이라 API/DSL로는 지정할 수 없다(create_view/update_view DSL에 COLOR 디렉티브 없음). 카드 자체 색이 필요하면 노션에서 보드 -> ... -> 레이아웃 -> 카드 색상을 "중요도"로 수동 지정한다.

[5] 마무리 규칙
- 상태는 select 타입(status 아님), 진도율은 percent 형식, 코어여부는 checkbox.
- 과목을 학습하면 상태를 "진행 중" -> "완료"로, 진도율을 갱신한다.
- 완료 후, 만든 데이터베이스의 URL과 과목 수(49개), 뷰 목록을 보고한다.
````

- 사전 준비: 웹 검색이 되는 Claude(WebSearch/WebFetch) + 레포 파일 쓰기.
- 참고: 기존 학습 노트(`year1/.../notes/*_학습노트.md`)가 출력 형식 예시. 파이프라인 1단계(조사·검증)와 2단계(마크다운 생성)를 함께 수행한다.

## 3. 텔레그램 노티 콘텐츠 생성

- 무엇을 만드나: 위 학습 노트를 매일 한 단계씩 따라가는 텔레그램 마이크로러닝 콘텐츠(`contents.md`)와 진도 파일(`progress.json`).
- 사전 준비: 해당 과목 학습 노트(#2 산출물) + 레포 파일 쓰기. 텔레그램 발송 스케줄은 별도(`~/.claude/scheduled-tasks/knou-daily-study-noti`).
- 핵심: 학습 노트를 "정본"으로 삼아, 훑어보며 따라가기 좋은 한 입 크기 단계로 재구성한다.

````text
너는 KNOU 신입생의 "매일 텔레그램 마이크로러닝" 콘텐츠 편집자다. 과목 "{과목명}"의 노티 콘텐츠를 만든다. 한국어, em dash(-) 금지.

[0] 정본 읽기: year{학년}/{학년}-{학기}_{과목명}/notes/{과목명}_학습노트.md 를 Read로 읽어 그 내용을 근거로 삼는다(사실/코드 정확 반영).

[1] 마이크로러닝 구성 원칙 (반드시 지킨다)
- 한 입 크기: 한 Step = 하루에 부담 없이 소화할 단일 개념/기능. 너무 무거우면 여러 Step으로 나누고, 너무 얇으면 합친다.
- 명료한 핵심 + 구체적 실습/예시 + 그날 할 작은 미션 + 현실적 예상 시간.
- 난이도 점진성: 선행 개념 위에 쌓이도록 순서를 배치하고 "앞 단계에서 배운 ~"로 연결, 능동 회상(직접 해보기)을 유도.

[2] 콘텐츠 작성 (Write로 저장)
경로: resources/study-noti/{과목명}/contents.md
- 파일은 "## Step 1:"로 바로 시작(머리말 없이). 학습 순서대로 충분한 수의 Step으로 분할(보통 18~30개, 내용량에 맞게).
- 각 Step은 정확히 아래 양식(빈 줄 포함)을 반복한다:

## Step 1: 단계 제목

📘 핵심
(2~5줄 핵심 개념)

💡 실습
(짧은 코드 + 기대 출력. 이론 과목이면 "💡 예시"로 바꾸고 구체 예시/비교)

🎯 미션
(작고 구체적인 그날의 과제)

⏱️ 예상 시간: NN분

✅ 마치면 텔레그램에서 "다음 {과목명}"

---

- 모든 Step에 위 양식 반복. 1부터 빈틈없이 연속 번호. 마지막 Step 뒤에도 "---"로 끝낸다.
- 마무리 문구는 한 글자도 다르지 않게: ✅ 마치면 텔레그램에서 "다음 {과목명}"

[3] 진도 파일 작성 (Write로 저장, UTF-8 + CRLF)
경로: resources/study-noti/{과목명}/progress.json
내용:
{
  "track": "{과목명} 학습",
  "label": "{과목명}",
  "keyword": "{과목명}",
  "currentStep": 1,
  "totalSteps": <작성한 Step 개수>,
  "completedSteps": [],
  "lastTelegramUpdateId": 0,
  "active": false,
  "notionUrl": "",
  "note": "active=true일 때만 노티 발송(학습 시작 시 true로). 진도 전진=텔레그램에 '다음 {과목명}' 텍스트를 보내면 currentStep+1. 회차는 노션 회차 토글로 사용자가 직접 관리."
}

[규칙] 한국어, em dash 금지, UTF-8(BOM 없음) + CRLF, 파일 끝 빈 줄. contents.md의 "## Step" 개수와 progress.json의 totalSteps가 반드시 일치해야 한다. 마무리 푸터의 과목명은 정확히 "{과목명}".
완료 후: contents.md Step 개수, progress.json totalSteps(일치 여부), 주요 단계 목록, 주요 개선점을 보고한다.
````

## 4. 노션 학습 트래킹 과목 양식 생성

- 무엇을 만드나: 위 노티(`contents.md`)의 Step 목록을 그대로 옮긴, 노션 트래커의 해당 과목 페이지 본문(회차 학습 양식).
- 사전 준비: Notion 연결 + #1로 만든 트래커 DB + 해당 과목 `contents.md`(#3 산출물).
- 핵심: contents.md의 Step을 "정본"으로 삼아 노션 토글을 채우고, 노티-노션을 링크로 연결한다.

````text
너는 Notion이 연결된 어시스턴트다. "방송대 컴퓨터과학과 학습 트래커" DB의 "{과목명}" 페이지 본문을 회차 학습 양식으로 채운다. 한국어, em dash(-) 금지.

[0] 정본 읽기: resources/study-noti/{과목명}/contents.md 를 Read로 읽는다. "## Step N: 제목" 목록이 Step의 정본이다(개수/순서/제목을 그대로 보존, 임의로 줄이거나 바꾸지 않는다).

[1] 대상 페이지 찾기
- 트래커 DB(방송대 컴퓨터과학과 학습 트래커)에서 과목명 = "{과목명}" 페이지를 찾는다. 없으면 이 문서 1번 "노션 학습 트래커 생성" 프롬프트의 속성/데이터 규칙대로 그 과목 행을 먼저 추가한다.
- 주의: Notion이 대시 없는 ID로 조회할 때 인접 페이지를 반환하는 글리치가 있으니, search로 정확한 페이지를 확인한 뒤 그 페이지에만 쓴다.

[2] 본문을 회차 양식으로 교체 (replace_content)
- 맨 위 안내 인용구 2줄:
  > 매일 텔레그램 노티로 받은 Step을 학습한 뒤, 해당 Step 토글 안에 "N회차" 토글을 만들어 그날 학습 내용을 자유롭게 정리하세요. 한 Step은 여러 번 복습하므로 회차를 늘려가며 기록합니다.
  > 상세 개념과 자료는 레포 학습 노트(year{학년}/{학년}-{학기}_{과목명}/notes/{과목명}_학습노트.md)를 참조하세요.
- 그 아래 "---" 구분선
- contents.md의 Step들을 학습 노트의 단원(또는 주차)을 기준으로 몇 개의 "## 그룹 헤딩"으로 묶는다(전체 Step을 빈틈없이 포함).
- 각 Step을 아래 토글로 변환한다. "## Step N: 제목"의 콜론 ": "을 가운뎃점 " · "으로만 바꿔 "Step N · 제목"으로:

<details>
<summary>Step 1 · 단계 제목</summary>
	<details>
	<summary>1회차</summary>
	</details>
</details>

- Step 개수/순서/제목을 contents.md와 정확히 일치시킨다(빠짐 없이).

[3] 노티-노션 연결
- 교체한 페이지의 URL을 resources/study-noti/{과목명}/progress.json 의 "notionUrl"에 기록한다(노티 메시지 하단에 이 링크가 붙는다).

토글 주의사항: Notion 토글은 HTML <details><summary>제목</summary> 내용 </details> 형태로만 렌더된다. <summary> 다음 줄에는 빈 줄 없이 탭으로 들여쓴 "1회차" <details>를 바로 붙인다. 화살표나 "-" 들여쓰기 방식은 토글이 되지 않는다.
완료 후: 페이지 URL, 토글(Step) 개수, contents.md와 일치 여부, progress.json notionUrl 갱신 여부를 보고한다.
````
