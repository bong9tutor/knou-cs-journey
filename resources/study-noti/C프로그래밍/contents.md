## Step 2: 첫 프로그램과 컴파일 4단계

📘 핵심
앞 단계에서 만든 환경으로 첫 C 프로그램을 돌려본다. C 소스가 실행되기까지 전처리 -> 컴파일 -> 링크 -> 실행 4단계를 거친다. main이 시작점이고 중괄호 { } 안이 함수 몸체이며, 모든 문장은 세미콜론(;)으로 끝난다.

💡 실습
```c
#include <stdio.h>
int main(void) {
    printf("Hello, C!\n");
    return 0;
}
```
컴파일/실행: gcc hello.c -o hello 후 ./hello (Windows는 hello.exe)
기대 출력: Hello, C!

🎯 미션
위 코드를 hello.c로 저장해 컴파일하고 실행해, "Hello, C!"가 출력되는지 확인하기.

⏱️ 예상 시간: 30분

## Step 3: 자료형과 변수

📘 핵심
변수는 값을 담는 메모리에 붙인 이름이며 자료형과 함께 선언한다. 정수 int, 실수 double, 문자 char가 기본이다. sizeof로 자료형이 차지하는 바이트 크기를 확인할 수 있다.

## Step 4: 표준 입출력 (printf / scanf)

📘 핵심
printf는 화면 출력, scanf는 키보드 입력이다. %d(정수) %f(실수) %c(문자) %s(문자열) 같은 서식 지정자로 값을 연결한다. scanf의 변수 앞에는 반드시 &(주소)를 붙인다(이유는 포인터 단원에서 이해하게 된다).

💡 실습
```c
int a, b;
scanf("%d %d", &a, &b);
printf("합: %d\n", a + b);
```
입력 7 5 -> 기대 출력: 합: 12

🎯 미션
두 정수를 입력받아 합과 곱을 출력하는 계산기 만들기.

⏱️ 예상 시간: 30분

## Step 5: 산술/관계/논리 연산자

📘 핵심
산술(+ - * / %), 관계(> == !=), 논리(&& || !) 연산자가 있다. 정수끼리 나눗셈은 몫만 남고(7/2=3), %는 나머지다. =(대입)와 ==(비교)는 전혀 다르니 조건문에서 절대 헷갈리지 않는다.

💡 실습
```c
int a = 7, b = 3;
printf("%d %d\n", a / b, a % b); // 2 1
printf("%d\n", a > b && b > 0);  // 1
```
기대 출력: 2 1 / 1

🎯 미션
정수 두 개의 사칙연산(+ - * /)과 나머지(%)를 모두 한 줄씩 출력해 보기.

⏱️ 예상 시간: 25분

## Step 6: 대입/증감 연산자와 단축 평가

📘 핵심
앞 단계 연산자에 이어, +=처럼 줄여 쓰는 대입과 ++/--(증감)를 익힌다. 후위 a++는 먼저 쓰고 증가, 전위 ++a는 먼저 증가 후 쓴다. 논리 &&, ||는 단축 평가라서 앞이 결과를 정하면 뒤를 보지 않는다.

💡 실습
```c
int a = 5;
printf("%d\n", a++); // 5 (출력 후 a는 6)
printf("%d\n", ++a); // 7 (먼저 7로 만든 뒤 출력)
```
기대 출력: 5 / 7

🎯 미션
변수 하나를 후위(a++)와 전위(++a)로 각각 출력해 차이를 눈으로 확인하기.

⏱️ 예상 시간: 25분

## Step 7: 형변환

📘 핵심
다른 자료형이 섞이면 작은 형이 큰 형으로 자동 변환된다. (double)처럼 명시적 형변환도 가능하다. 평균 계산에서 정수 나눗셈 함정을 (double) 캐스팅으로 피한다.

💡 실습
```c
int total = 7, count = 2;
printf("%.1f\n", total / count);          // 3.0
printf("%.1f\n", (double)total / count);  // 3.5
```
기대 출력: 3.0 / 3.5

🎯 미션
정수 점수 3개의 평균을 소수점 한 자리까지 정확히 출력하기.

⏱️ 예상 시간: 25분

## Step 8: 조건문 (if / else)

📘 핵심
if / else if / else로 조건에 따라 분기한다. 위에서부터 차례로 검사하므로 범위가 큰 조건을 먼저 두면 안 된다(예: score>=70을 score>=90보다 먼저 두면 안 됨).

```c
else printf("C\n");
```
score가 85면 기대 출력: B

## Step 9: switch 문과 삼항 연산자

📘 핵심
switch는 정수나 문자 하나를 여러 값과 비교할 때 깔끔하다. 각 case 끝에 break를 빠뜨리면 다음 case로 흘러간다(fall-through). 삼항 연산자 조건 ? 참값 : 거짓값 은 간단한 분기를 한 줄로 줄인다.

💡 실습
```c
char g = 'B';
switch (g) {
    case 'A': printf("훌륭\n"); break;
    case 'B': printf("좋음\n"); break;
    default:  printf("분발\n");
}
```
기대 출력: 좋음

🎯 미션
학점 문자(A~F)를 입력받아 switch로 한 줄 평가를 출력하기.

⏱️ 예상 시간: 30분

## Step 10: 반복문 (for / while)

📘 핵심
for는 반복 횟수가 정해졌을 때, while은 조건이 참인 동안 반복한다. do-while은 조건을 나중에 검사해 최소 한 번 실행한다. break는 반복 종료, continue는 이번 회차 건너뛰기다.

💡 실습
```c
int sum = 0;
for (int i = 1; i <= 10; i++) sum += i;
printf("%d\n", sum); // 55
```
기대 출력: 55

🎯 미션
1부터 100까지 정수의 합을 for와 while 두 방법으로 각각 구해 같은 값이 나오는지 확인하기.

## Step 11: 중첩 반복문

📘 핵심
반복문 안에 반복문을 넣어 표 형태를 만든다. 바깥 반복이 한 번 돌 때 안쪽 반복이 통째로 돈다. 구구단, 별 찍기, 2차원 데이터 처리에 쓰인다.

💡 실습
```c
for (int i = 2; i <= 9; i++) {

        printf("%d*%d=%d ", i, j, i*j);
    printf("\n");
}
```
기대 출력: 2단부터 9단까지 한 줄씩 8줄

🎯 미션

## Step 13: 변수의 범위와 static

📘 핵심
함수 안 변수(지역)는 그 함수에서만 산다. 함수 밖 변수(전역)는 어디서나 쓰인다. static 지역 변수는 함수가 끝나도 값이 유지되어 호출 사이에 상태를 기억한다.

## Step 14: 배열

📘 핵심
같은 자료형 값 여러 개를 하나의 이름으로 묶는다. 인덱스는 0부터 시작한다. C는 범위 검사를 안 하므로 크기 n 배열은 0~n-1만 접근해야 한다(off-by-one 주의).

💡 실습
```c
int score[5] = {90, 85, 70, 95, 60};
int sum = 0;
for (int i = 0; i < 5; i++) sum += score[i];
printf("평균 %.1f\n", (double)sum / 5);
```
기대 출력: 평균 80.0

🎯 미션
정수 5개를 배열에 담아 최댓값과 평균 구하기.

⏱️ 예상 시간: 35분

## Step 15: 2차원 배열

📘 핵심
행과 열을 가진 표 형태 데이터다. m[행][열]로 접근하고 중첩 반복으로 순회한다. 앞 단계의 중첩 반복문이 그대로 쓰인다. 성적표, 행렬, 격자 게임판 등에 활용한다.

💡 실습
```c
int m[2][3] = {{1,2,3}, {4,5,6}};
for (int i = 0; i < 2; i++) {
    for (int j = 0; j < 3; j++) printf("%d ", m[i][j]);
    printf("\n");
}
```
기대 출력: 1 2 3 / 4 5 6

🎯 미션
3x3 행렬을 만들어 모든 원소의 합 구하기.

⏱️ 예상 시간: 30분

## Step 16: 문자열

📘 핵심
C 문자열은 끝에 널 문자('\0')가 붙은 char 배열이다. <string.h>의 strlen(길이), strcpy(복사), strcat(연결), strcmp(비교)를 쓴다. 문자열 비교는 ==가 아니라 strcmp로 한다(==는 내용이 아닌 주소를 비교).

## Step 17: 포인터 기초

📘 핵심
포인터는 변수의 주소를 저장하는 변수다. &변수는 주소를 구하고(address-of), *포인터는 가리키는 값을 구한다(역참조). C의 핵심이니 변수와 주소를 그림으로 그려가며 천천히 익힌다.

💡 실습
```c
int a = 10;
int *p = &a;
printf("%d\n", *p); // 10
*p = 20;
printf("%d\n", a);  // 20
```
기대 출력: 10 / 20 (p를 통해 a가 바뀜)

🎯 미션
변수 하나를 포인터로 가리켜, 포인터를 통해 값을 20으로 바꾼 뒤 원래 변수도 바뀌었는지 확인하기.

⏱️ 예상 시간: 40분

## Step 18: 포인터로 함수 인자 전달

📘 핵심
주소를 넘기면 함수가 원래 변수를 직접 바꿀 수 있다(call by reference 흉내). 앞서 scanf에 &를 붙인 이유가 바로 이것이다. 두 값을 맞바꾸는 swap이 대표 예제다.

💡 실습
```c
void swap(int *x, int *y) {
    int t = *x; *x = *y; *y = t;
}
// swap(&a, &b); 로 a와 b가 실제로 바뀐다
```
기대 동작: a=1, b=2였다면 호출 후 a=2, b=1

🎯 미션
두 변수의 값을 맞바꾸는 swap 함수를 포인터로 구현해 결과 확인하기.

⏱️ 예상 시간: 40분

## Step 20: NULL 포인터와 안전한 사용

📘 핵심
아무것도 가리키지 않는 포인터는 NULL로 초기화한다. 초기화 안 한 포인터(쓰레기 주소)나 NULL을 역참조하면 프로그램이 죽는다(segmentation fault). 포인터는 쓰기 전에 유효한 곳을 가리키는지 확인하는 습관을 들인다.

💡 실습
```c
int a = 5;
int *p = NULL;        // 안전한 초기값
p = &a;               // 유효한 곳을 가리키게 한 뒤
if (p != NULL) printf("%d\n", *p); // 검사 후 사용
```
기대 출력: 5

🎯 미션
NULL로 초기화한 포인터를, NULL 검사를 거친 뒤에만 역참조해 값을 출력하기.

⏱️ 예상 시간: 25분

## Step 22: 구조체 포인터와 화살표 연산자

📘 핵심
구조체 포인터에서 멤버 접근은 화살표(->)를 쓴다(p->age는 (*p).age와 같음). 함수에 구조체를 넘길 때 통째로 복사하면 느리므로 보통 포인터로 넘겨 원본을 직접 다룬다.

💡 실습
```c

void move(Point *p) { p->x += 1; p->y += 1; }
// move(&pt); 로 pt가 직접 바뀐다
```
기대 동작: pt가 (1,1)이면 호출 후 (2,2)

🎯 미션
Point 구조체 포인터를 함수에 넘겨 좌표를 (+1, +1) 이동시키기.

## Step 23: 공용체 (union)

📘 핵심
공용체는 구조체와 문법이 같지만 모든 멤버가 같은 메모리를 공유해 한 번에 하나만 의미가 있다. 크기는 가장 큰 멤버에 맞춰진다. 메모리를 아끼거나 같은 데이터를 다른 형으로 해석할 때 쓴다.

💡 실습
```c
union Data { int i; float f; };
union Data d;
d.i = 65;  printf("%d\n", d.i);      // 65
d.f = 3.14f; printf("%.2f\n", d.f);  // 3.14 (같은 공간을 덮어씀)
```
기대 출력: 65 / 3.14

🎯 미션
int와 float을 멤버로 갖는 union을 만들어, 값을 덮어쓰며 sizeof로 크기를 출력해 보기.

⏱️ 예상 시간: 25분

## Step 24: 동적 메모리 할당 (malloc / free)

📘 핵심
실행 중에 필요한 만큼 메모리를 얻으려면 <stdlib.h>의 malloc을 쓰고, 다 쓰면 반드시 free로 반납한다. malloc은 실패 시 NULL을 주므로 항상 검사한다. 누수와 double free를 조심한다.

💡 실습
```c
int *arr = malloc(n * sizeof(int));
if (arr == NULL) return 1;
// ... 사용 ...
free(arr);
arr = NULL;   // 댕글링 포인터 방지
```
기대 동작: n개 정수를 동적 배열로 다룸

🎯 미션
사용자가 입력한 개수만큼 동적 배열을 만들어 값을 채우고 합 구한 뒤 free 하기.

⏱️ 예상 시간: 45분

## Step 25: calloc과 realloc

📘 핵심
앞 단계의 malloc이 준 메모리에는 쓰레기 값이 들어 있다. 0으로 채워진 메모리가 필요하면 calloc(개수, 한개크기), 이미 할당한 블록을 늘리거나 줄이려면 realloc을 쓴다. realloc 결과는 임시 변수로 받아 성공을 확인한 뒤 교체하는 것이 안전하다.

💡 실습
```c
int *arr = calloc(3, sizeof(int)); // 0 0 0
int *tmp = realloc(arr, 5 * sizeof(int));
if (tmp != NULL) arr = tmp;        // 성공 시 교체
```
기대 동작: 0으로 초기화된 3칸을 5칸으로 확장(기존 값 보존)

🎯 미션
calloc으로 3칸을 0으로 만들어 출력하고, realloc으로 5칸으로 늘려 값을 채워 출력하기.

⏱️ 예상 시간: 35분

## Step 26: 파일 입출력 (텍스트)

📘 핵심
FILE * 포인터로 파일을 다룬다. 순서는 fopen(열기) -> 읽기/쓰기 -> fclose(닫기)다. 모드는 "r"(읽기) "w"(쓰기) "a"(이어쓰기). fprintf/fscanf로 서식 입출력한다. fopen 실패 시 NULL이므로 항상 검사한다.

💡 실습
```c
FILE *fp = fopen("data.txt", "w");
if (fp == NULL) return 1;
fprintf(fp, "홍길동 90\n");
fclose(fp);
```
기대 동작: 같은 폴더에 data.txt가 생기고 "홍길동 90"이 들어감

🎯 미션
이름과 점수를 파일에 저장한 뒤 다시 "r"로 열어 읽어 화면에 출력하기.

⏱️ 예상 시간: 45분

## Step 27: 이진 파일과 랜덤 접근

📘 핵심
구조체나 배열을 바이트 그대로 저장하면 이진(binary) 파일이다. 모드 뒤에 b를 붙여("wb","rb") 연다. fwrite/fread로 레코드를 통째 읽고 쓰며, 레코드 크기가 고정이라 fseek으로 N번째 레코드 위치로 곧장 점프하는 랜덤 접근이 가능하다.

💡 실습
```c
fwrite(list, sizeof(struct Student), 3, fp); // 3개 기록
fseek(fp, 1 * sizeof(struct Student), SEEK_SET); // 2번째로 이동
fread(&one, sizeof(struct Student), 1, fp);
```
기대 동작: 두 번째 레코드만 콕 집어 읽음

🎯 미션
Student 구조체 3개를 "wb"로 저장한 뒤, fseek으로 2번째 레코드만 읽어 출력하기.

⏱️ 예상 시간: 40분

## Step 28: 전처리기와 매크로

📘 핵심
#으로 시작하는 줄은 컴파일 전에 전처리된다. #include는 헤더 포함, #define은 매크로다. 함수형 매크로는 인자와 전체를 괄호로 감싼다(안 그러면 SQUARE(1+2)가 잘못 펼쳐짐). #ifndef/#define/#endif는 인클루드 가드다.

💡 실습
```c
#define PI 3.14159
#define SQUARE(x) ((x) * (x))
printf("%.2f\n", PI * SQUARE(2.0)); // 12.57
```
기대 출력: 12.57

🎯 미션
원의 넓이를 구하는 매크로를 정의해 반지름 3의 넓이를 출력하기.

⏱️ 예상 시간: 30분

## Step 29: 재귀 함수

📘 핵심
함수가 자기 자신을 호출하는 것이 재귀다. 반드시 종료 조건(base case)이 있어야 무한 호출을 멈춘다. 팩토리얼, 피보나치가 대표 예제다. 종료 조건을 빠뜨리면 스택 오버플로로 죽는다.

💡 실습
```c
int fact(int n) {
    if (n <= 1) return 1;
    return n * fact(n - 1);
}
// fact(5) == 120
```
기대 출력: 120

🎯 미션
팩토리얼을 재귀로 구현하고, 같은 것을 반복문으로도 만들어 결과가 같은지 비교하기.

⏱️ 예상 시간: 40분

## Step 30: 다중 파일 구성과 디버깅

📘 핵심
큰 프로그램은 선언을 헤더(.h), 정의를 소스(.c)로 나눈다. 내 헤더는 "큰따옴표"로 포함하고 인클루드 가드를 둔다. 컴파일은 gcc -Wall -Wextra -g로 경고를 켜고, 메모리 오류는 valgrind로 찾는다.

💡 실습
```c
// mymath.h
#ifndef MYMATH_H
#define MYMATH_H
int add(int a, int b);
#endif
// 컴파일: gcc main.c mymath.c -o app
```
기대 동작: 두 소스를 함께 컴파일해 하나의 실행 파일 생성

🎯 미션
add/mul 함수를 헤더와 소스로 분리해 main.c에서 호출하는 프로그램 완성하기.

⏱️ 예상 시간: 45분
