컴퓨터과학개론

컴퓨터과학 전 분야의 용어와 원리를 한 번씩 훑어 상위 전공(자료구조-운영체제-컴퓨터구조-데이터베이스-정보통신망)의 토대를 만드는 과목이다. 한 분야를 깊게 파기보다, 전체 지도를 그려 두는 데 목표를 둔다.

1. 컴퓨터과학이란 무엇인가

개요 - 계산과 정보 처리의 학문

컴퓨터과학(Computer Science)은 "계산(computation)"과 "정보 처리"를 연구하는 학문이다. 흔히 컴퓨터라는 기계를 다루는 학문으로 오해하지만, 본질은 문제를 어떻게 표현하고, 어떤 절차로 풀며, 그 절차가 얼마나 효율적인가를 다룬다. 컴퓨터는 그 절차를 실행하는 도구일 뿐이다.

영역다루는 질문방통대 대응 과목
이론무엇을 계산할 수 있는가, 얼마나 빠른가알고리즘, 컴파일러구성
시스템하드웨어/소프트웨어가 어떻게 동작하는가컴퓨터구조, 운영체제, 정보통신망
데이터정보를 어떻게 저장-조직-검색하는가자료구조, 데이터베이스시스템
응용사람의 문제를 어떻게 푸는가인공지능, 소프트웨어공학

컴퓨터과학 전체를 관통하는 세 가지 핵심 사고법:

  • 알고리즘(algorithm): 문제를 푸는 단계의 명확한 나열. "유한한 단계 안에 끝나고, 입력에 대해 정해진 출력을 낸다"는 조건을 만족해야 한다.
  • 추상화(abstraction): 복잡한 대상에서 지금 필요한 본질만 남기고 세부를 감추는 것. 가장 중요한 사고법이다. (예: 운전자는 엔진 내부를 몰라도 페달과 핸들이라는 추상화된 인터페이스로 차를 몬다.)
  • 자동화(automation): 추상화한 절차를 기계가 반복 실행하게 만드는 것.
1.1 컴퓨터과학 vs 컴퓨터공학

비슷해 보이지만 강조점이 다르다. 둘은 겹치는 영역이 넓고 상호 보완적이다.

뿌리수학-논리학전기전자공학

방통대 컴퓨터과학과는 이름 그대로 소프트웨어-이론 중심이지만, 디지털논리회로-컴퓨터구조처럼 하드웨어 기초도 함께 다룬다.

1.2 컴퓨터 시스템의 4가지 구성요소

컴퓨터 "시스템"은 기계(하드웨어)만이 아니라 다음 네 가지가 맞물려 동작한다.

  • 하드웨어(hardware): CPU-기억장치-입출력장치 등 물리적 장치.
  • 소프트웨어(software): 하드웨어를 동작시키는 프로그램. 시스템 소프트웨어(OS 등)와 응용 소프트웨어로 나뉜다.
  • 데이터(data): 프로그램이 처리하는 입력-출력-저장 정보. 가공되어 의미를 가지면 정보(information)가 된다.
  • 사용자(user): 시스템을 사용-운영하는 사람. 일반 사용자부터 시스템 관리자-개발자까지 포함한다.

본질은 문제를 어떻게 표현하고 어떤 절차로 풀며 그 절차가 얼마나 효율적인가를 다루는 데 있다. 컴퓨터는 그 절차를 실행하는 도구일 뿐이다. 핵심 사고법은 알고리즘-추상화-자동화다.

2. 컴퓨터의 역사와 발전

기계식 계산기에서 폰 노이만 구조까지

기계로 계산을 자동화하려는 시도는 수백 년에 걸쳐 발전했다.

  • 기계식 계산기: 파스칼의 가산기(17세기), 라이프니츠의 사칙연산기. 톱니바퀴로 숫자를 다뤘다.
  • 해석기관(Analytical Engine): 찰스 배비지가 19세기에 구상한, 프로그램으로 제어되는 범용 계산기 개념. 에이다 러브레이스가 최초의 알고리즘을 기술해 "최초의 프로그래머"로 불린다.
  • 튜링 기계(Turing Machine, 1936): 앨런 튜링이 제안한 추상적 계산 모델. "계산 가능하다"는 것의 수학적 정의를 세워 현대 컴퓨터의 이론적 뿌리가 되었다.
  • 폰 노이만 구조(1945): 프로그램과 데이터를 같은 메모리에 저장하는 프로그램 내장 방식(stored-program). 오늘날 거의 모든 컴퓨터가 이 구조를 따른다.
컴퓨터 세대 구분과 무어의 법칙

소자(부품)를 기준으로 세대를 나눈다.

무어의 법칙(Moore's Law)

집적회로의 트랜지스터 수가 약 2년마다 두 배가 된다는 경험 법칙. 수십 년간 성능 향상의 동력이었으나 물리적 한계로 둔화되며, 그 대안으로 멀티코어-병렬처리-GPU가 부상했다.

3. 진법과 데이터 표현

비트, 바이트, 데이터 단위

컴퓨터는 전기의 켜짐/꺼짐 두 상태만 구분한다. 그래서 모든 정보를 2진수(binary), 즉 0과 1의 조합으로 표현한다. 정보의 최소 단위가 비트(bit)이고, 8비트가 모여 바이트(byte)가 된다.

데이터 단위 스펙트럼(작은 것 -> 큰 것):

비트(bit) -> 바이트(B, 8bit) -> KB -> MB -> GB -> TB -> PB -> EB -> ZB -> YB
  • 1바이트씩 올라갈 때마다 2진 기준 1,024배(= 2^10)씩 커진다. KB(킬로)-MB(메가)-GB(기가)-TB(테라)-PB(페타)-EB(엑사)-ZB(제타)-YB(요타) 순.
  • 1바이트(8비트)는 2^8 = 256가지 상태를 표현할 수 있어 ASCII 문자 한 글자나 0~255 정수를 담는 기본 단위로 쓰인다.
3.1 진법 변환

r진법(base-r)의 일반 정의: r진법에서 한 자리에는 0부터 r-1까지 r개의 숫자(digit)를 쓰고, 각 자리의 가중치는 밑수 r의 거듭제곱이다. n자리 정수 값은 다음과 같다.

값 = d_(n-1) x r^(n-1) + ... + d_1 x r^1 + d_0 x r^0

예: 2진법 r=2(0,1), 8진법 r=8(0~7), 10진법 r=10(0~9), 16진법 r=16(0~9, A~F). 16진수 1A = 1 x 16^1 + 10 x 16^0 = 26.

10진수2진수8진수16진수
10101012A
15111117F
16100002010
25511111111377FF
  • 10진 -> 2진: 2로 계속 나누며 나머지를 거꾸로 읽는다. 13 = 2x6+1, 6=2x3+0, 3=2x1+1, 1=2x0+1 -> 아래에서 위로 1101.
  • 2진 -> 10진: 자리값(2의 거듭제곱)을 곱해 더한다. 1101 = 8+4+0+1 = 13.
  • 16진수는 4비트를 한 자리로 묶어 표기하므로 2진수와 변환이 쉽다(예: 1101 1110 = DE).
3.2 정수 표현 - 4가지 방식

고정된 n비트로 정수를 표현하는 방식은 크게 네 가지다. 앞의 셋은 음수까지 담고, 부호없는 정수는 0 이상만 담는다.

방식음수 표현 방법4비트 +5 / -50의 표현n비트 범위
부호없는 정수음수 없음(0 이상만)+5 = 0101 (-5 없음)0000 (1개)0 ~ 2^n - 1
부호화-크기최상위 비트=부호, 나머지=절댓값0101 / 1101+0=0000, -0=1000 (2개)-(2^(n-1)-1) ~ +(2^(n-1)-1)
1의 보수모든 비트 반전(0<->1)0101 / 1010+0=0000, -0=1111 (2개)-(2^(n-1)-1) ~ +(2^(n-1)-1)
2의 보수1의 보수에 1을 더함0101 / 10110000 (1개)-2^(n-1) ~ +2^(n-1) - 1
  • 부호없는 정수: 부호 비트가 없어 같은 비트 수로 가장 큰 양수까지 담는다. 8비트면 0~255.
  • 부호화-크기: 가장 직관적이다. 최상위 1비트로 부호(0=양, 1=음), 나머지로 크기. 단점은 +0과 -0이 따로 존재하고 덧셈 회로가 복잡해진다는 것.
  • 1의 보수: 모든 비트를 반전해 음수를 만든다. +5 = 0101 -> -5 = 1010. 역시 +0(0000)과 -0(1111)이 둘 다 존재한다.
  • 2의 보수: 1의 보수에 1을 더한다. +5 = 0101 -> 반전 1010 -> +1 = 1011 = -5. 0이 하나뿐(0000)이고 뺄셈을 덧셈 회로로 그대로 처리할 수 있다.

현대 표준은 2의 보수

0이 하나뿐이고 덧셈 회로를 그대로 쓸 수 있어 현대 컴퓨터의 표준이다. n비트 2의 보수 범위는 -2^(n-1) ~ +2^(n-1)-1 (8비트면 -128 ~ +127).

3.3 실수 표현 - 부동소수점

아주 크거나 작은 실수를 고정된 비트로 표현하려고 부호 + 지수 + 가수(mantissa)로 나눈다. 과학적 표기법(예: 1.01 x 2^3)을 2진수로 구현한 것이다. 국제 표준 IEEE 754가 32비트(단정밀도)/64비트(배정밀도)를 정의한다.

0.1 + 0.2 != 0.3

0.1처럼 10진 소수는 2진 부동소수점으로 정확히 표현되지 않아 미세한 오차가 생긴다. 금액 계산에 부동소수점을 쓰면 안 되는 근거다.

3.4 문자 표현 - 문자 코드
  • ASCII: 7비트로 영문자-숫자-기호 128개를 표현. 'A'=65, 'a'=97, '0'=48.
  • EUC-KR / CP949: 한글을 2바이트로 표현하던 한국형 코드.
  • 유니코드(Unicode): 전 세계 문자를 하나의 번호 체계로 통합. 실제 저장 인코딩으로 UTF-8(가변 길이, 영문 1바이트-한글 3바이트, 웹 표준)이 가장 널리 쓰인다.

"뷁" 현상의 원인

파일이 깨져 보이는 현상은 대부분 저장한 인코딩과 읽는 인코딩이 달라서 생긴다. 이 레포가 UTF-8을 강제하는 이유이기도 하다.

  • 2의 보수에서 +5(0101)의 -5는?
    반전 1010에 1을 더해 1011. 0이 하나뿐이고 뺄셈을 덧셈으로 처리.
  • 1바이트(8비트)가 표현하는 상태 수는?
    2^8 = 256가지. ASCII 한 글자나 0~255 정수를 담는다.
  • 웹 표준 문자 인코딩은?
    UTF-8 (가변 길이, 영문 1바이트-한글 3바이트). 유니코드를 저장하는 방식.

4. 불 대수와 논리 회로

보강 단원 시험 필수 범위 아님 - 상위과목(디지털논리회로-컴퓨터구조) 대비 보강.

4.1 기본 논리 게이트

컴퓨터 하드웨어는 0과 1의 논리 연산으로 모든 계산을 수행한다. 그 수학적 토대가 불 대수(Boolean algebra)다.

게이트의미예 (A=1, B=0)
AND둘 다 1이면 11 AND 0 = 0
OR하나라도 1이면 11 OR 0 = 1
NOT반전NOT 1 = 0
NANDAND의 반전1 NAND 0 = 1
NOROR의 반전1 NOR 0 = 0
XOR다르면 11 XOR 0 = 1

NAND(또는 NOR) 게이트 하나만으로 다른 모든 게이트를 만들 수 있어 "완전 집합(universal gate)"이라 부른다.

4.2 불 대수 법칙
  • 교환/결합/분배 법칙은 일반 대수와 유사하다.
  • 드모르간의 법칙: NOT(A AND B) = (NOT A) OR (NOT B), NOT(A OR B) = (NOT A) AND (NOT B). 회로 단순화의 핵심 도구.
4.3 조합 회로와 순차 회로
  • 조합 회로(combinational): 현재 입력만으로 출력이 정해짐. 가산기(adder), 멀티플렉서, 디코더 등. 1비트 덧셈을 하는 반가산기는 합(XOR)과 자리올림(AND)으로 만든다. 자리올림 입력까지 받는 것이 전가산기.
  • 순차 회로(sequential): 과거 상태(기억)에 따라 출력이 달라짐. 1비트를 저장하는 플립플롭(flip-flop)이 기본 단위이며, 이것이 모여 레지스터와 메모리가 된다.
Q. 게이트 하나만으로 다른 모든 게이트를 만들 수 있는 "완전 집합" 게이트는?
정답 보기
NAND 또는 NOR. 이 둘 각각으로 AND/OR/NOT을 모두 구성할 수 있다.

5. 컴퓨터 구조 - CPU, 메모리, 입출력

5.1 CPU의 구성

폰 노이만 구조의 컴퓨터는 크게 중앙처리장치(CPU)-기억장치(memory)-입출력장치(I/O)로 이뤄지고, 이들을 버스(bus)가 연결한다.

  • 산술논리장치(ALU): 실제 사칙연산-논리연산 수행.
  • 제어장치(CU): 명령어를 해석해 각 부품에 신호를 보냄.
  • 레지스터(register): CPU 내부의 초고속 임시 저장소. 다음 명령 주소를 가리키는 프로그램 카운터(PC), 현재 명령을 담는 명령 레지스터(IR) 등.
5.2 명령어 실행 사이클

CPU는 다음을 끊임없이 반복한다.

  1. 인출(Fetch): PC가 가리키는 메모리에서 명령어를 가져옴.
  2. 해독(Decode): 제어장치가 명령어 종류를 해석.
  3. 실행(Execute): ALU 등이 연산 수행.
  4. 저장(Store/Write-back): 결과를 레지스터/메모리에 기록.

성능 향상 기법: 파이프라이닝(여러 명령의 각 단계를 겹쳐 실행), 멀티코어(CPU 코어를 여러 개), 캐시(자주 쓰는 데이터를 CPU 가까이에).

5.3 기억장치 계층

위로 갈수록 빠르고 비싸고 작다. 아래로 갈수록 느리고 싸고 크다.

계층속도용량휘발성
레지스터가장 빠름수십 바이트휘발성
캐시(L1/L2/L3)매우 빠름KB~MB휘발성
주기억장치(RAM)빠름GB휘발성
보조기억장치(SSD/HDD)느림TB비휘발성
  • RAM: 전원이 꺼지면 내용이 사라짐(휘발성), 프로그램 실행 중 데이터를 담는 작업 공간.
  • ROM: 읽기 전용, 부팅 같은 기본 동작을 담음.
  • 클럭(clock): CPU의 동작 박자. 3GHz는 초당 30억 번 박자.
5.4 입출력(I/O)

키보드-마우스-모니터-디스크 등 외부 장치와 데이터를 주고받는다. 효율을 위해 다음 기법을 쓴다.

  • 인터럽트(interrupt): 장치가 CPU에 "처리해 달라"고 신호.
  • DMA(직접 메모리 접근): CPU를 거치지 않고 장치-메모리 직접 전송.
Q. 명령어 실행 사이클의 4단계를 순서대로 말해 보자.
정답 보기
인출(Fetch) -> 해독(Decode) -> 실행(Execute) -> 저장(Store/Write-back).

6. 운영체제 기초

운영체제의 핵심 역할 4가지

운영체제(OS)는 하드웨어와 응용 프로그램 사이에서 자원을 관리하고 사용자에게 편리한 환경을 제공하는 시스템 소프트웨어다. (Windows, Linux, macOS, Android, iOS 등)

역할설명핵심 기법
프로세스 관리실행 중인 프로그램에 CPU를 나눠 줌스케줄링: FCFS, SJF, 라운드 로빈
주기억장치 관리한정된 RAM을 프로세스들에 배분가상 메모리, 페이징
장치 관리입출력 장치를 제어드라이버
파일 관리데이터를 파일-디렉터리로 저장권한 관리
핵심 추가 개념
  • 프로세스 vs 스레드: 프로세스는 독립된 실행 단위(고유 메모리 공간), 스레드는 한 프로세스 안에서 메모리를 공유하며 동시에 도는 더 가벼운 실행 흐름.
  • 교착 상태(deadlock): 둘 이상의 프로세스가 서로의 자원을 기다리며 무한정 멈추는 상황.
  • 커널(kernel): OS의 핵심부. 하드웨어를 직접 다루며 가장 높은 권한으로 동작.

CPU 스케줄링 대표 알고리즘

FCFS(선착순), SJF(짧은 작업 우선), 라운드 로빈(시간 분할). 여러 프로세스가 동시에 도는 것처럼 보이게 CPU를 빠르게 전환한다.

  • 프로세스 vs 스레드의 메모리 차이는?
    프로세스는 고유 메모리 공간을 가진 독립 실행 단위, 스레드는 한 프로세스 안에서 메모리를 공유하는 더 가벼운 실행 흐름.
  • 물리 메모리보다 큰 주소 공간을 쓰게 하는 기법은?
    가상 메모리(virtual memory). 메모리를 일정 크기로 나누는 페이징(paging)과 함께 쓴다.

7.2 알고리즘과 복잡도(빅오 표기법)

알고리즘의 좋고 나쁨은 입력 크기 n이 커질 때의 자원 사용 증가율로 따진다. 이를 빅오 표기법(Big-O)으로 나타낸다.

표기이름예
O(1)상수배열 인덱스 접근
O(log n)로그이진 탐색
O(n)선형순차 탐색
O(n log n)선형로그병합-퀵 정렬(평균)
O(n^2)제곱버블-선택 정렬
O(2^n)지수부분집합 완전 탐색
  • 정렬: 버블-선택-삽입 정렬(O(n^2)), 병합-퀵 정렬(평균 O(n log n)).
  • 탐색: 순차 탐색(O(n)), 이진 탐색(정렬된 데이터에서 O(log n)).

8. 자료구조 개요

대표 자료구조 비교

자료구조(data structure)는 데이터를 메모리에 효율적으로 조직하는 방식이다. 어떤 자료구조를 쓰느냐가 알고리즘의 속도를 좌우한다.

자료구조특징대표 연산 / 용도
배열(array)같은 자료를 연속 공간에 저장, 인덱스로 즉시 접근임의 접근 O(1), 삽입/삭제 비쌈
연결 리스트(linked list)노드가 다음 노드를 가리킴삽입/삭제 유리, 임의 접근 느림
스택(stack)후입선출(LIFO)함수 호출, 실행 취소(undo), 괄호 검사
큐(queue)선입선출(FIFO)작업 대기열, 너비 우선 탐색
트리(tree)계층 구조, 부모-자식디렉터리, 이진 탐색 트리(탐색 O(log n))
그래프(graph)정점과 간선의 망지도 경로, SNS 관계, 네트워크
해시 테이블(hash table)키를 함수로 위치 변환평균 O(1) 검색, 사전(dict)
  • 스택과 큐의 차이: 스택은 쌓은 접시처럼 마지막에 넣은 것을 먼저 꺼내고, 큐는 줄 서기처럼 먼저 온 것을 먼저 꺼낸다.
  • 이진 탐색 트리: 왼쪽 자식 < 부모 < 오른쪽 자식 규칙으로, 균형이 잡히면 탐색이 O(log n)이다.
  • LIFO 자료구조와 FIFO 자료구조는?
    LIFO = 스택(마지막에 넣은 것 먼저), FIFO = 큐(먼저 온 것 먼저).
  • 평균 O(1) 검색을 제공하는 자료구조는?
    해시 테이블(hash table). 키를 해시 함수로 위치로 변환한다. 사전(dict)의 토대.

9. 컴퓨터 네트워크와 인터넷/웹

9.1 네트워크 기본

여러 컴퓨터를 연결해 데이터를 주고받는 것이 컴퓨터 네트워크다. 인터넷은 전 세계 네트워크를 하나로 묶은 "네트워크의 네트워크"다. 데이터는 작은 조각인 패킷(packet)으로 나뉘어 전송된다(패킷 교환).

규모별 분류: LAN(근거리, 사무실/가정), WAN(원거리, 도시-국가 간).

9.2 프로토콜 계층 - TCP/IP와 OSI

통신 규칙(프로토콜)을 역할별로 층으로 나눈다. 각 층은 아래층 서비스를 이용하고 위층에 서비스를 제공한다.

TCP/IP 4계층대표 프로토콜역할
응용 계층HTTP, FTP, SMTP, DNS사용자 서비스
전송 계층TCP, UDP종단 간 신뢰성 있는/빠른 전달
인터넷 계층IP주소(IP)로 경로 선택(라우팅)
네트워크 접근 계층Ethernet, Wi-Fi물리적 전송

이론적 참조 모델인 OSI 7계층은 TCP/IP 4계층을 더 잘게 쪼갠 것이다(위가 사용자에 가깝고, 아래가 물리에 가깝다).

OSI 7계층대응 TCP/IP역할대표 프로토콜/예
7. 응용응용 계층사용자 서비스-응용 인터페이스HTTP, FTP, SMTP, DNS
6. 표현응용 계층인코딩-암호화-압축TLS, JPEG, ASCII
5. 세션응용 계층연결(세션) 설정-유지-종료RPC, 세션 관리
4. 전송전송 계층종단 간 전달, 신뢰성-흐름 제어TCP, UDP
3. 네트워크인터넷 계층주소 지정-경로 선택(라우팅)IP, ICMP, 라우터
2. 데이터 링크네트워크 접근 계층인접 노드 간 프레임 전송-오류 검출Ethernet, MAC, 스위치
1. 물리네트워크 접근 계층비트를 전기-광-전파 신호로 전송케이블, 허브, 신호 규격
  • TCP vs UDP: TCP는 연결을 맺고 순서-재전송을 보장(웹-메일), UDP는 빠르지만 보장 없음(실시간 스트리밍-게임).
  • IP 주소와 DNS: IP 주소(예: 142.250.x.x)는 컴퓨터의 번지수. DNS는 사람이 외우는 도메인 이름(google.com)을 IP 주소로 변환하는 인터넷의 전화번호부. IPv4 주소 고갈로 IPv6로 전환 중.
9.3 웹의 동작
  • 웹(WWW)은 인터넷 위에서 도는 하나의 서비스. HTTP(S) 프로토콜로 통신한다.
  • URL: 자원의 주소(https://도메인/경로).
  • 클라이언트-서버 모델: 브라우저(클라이언트)가 요청(request)하면 서버가 응답(response)으로 HTML/CSS/JS를 보내고, 브라우저가 화면으로 렌더링한다.
  • HTTPS: HTTP에 암호화(TLS)를 더해 도청-위변조를 막은 안전한 통신.
Q. 실시간 게임이나 영상 스트리밍에 TCP보다 UDP를 쓰는 이유는?
정답 보기
UDP는 순서 보장-재전송 없이 빠르게 보낸다. 약간의 손실보다 지연이 더 치명적인 실시간 서비스에 적합하다. TCP는 신뢰성을 보장하는 대신 느리다.

10. 데이터베이스 기초

10.1 관계형 데이터베이스

데이터베이스(DB)는 여러 사용자가 공유하는, 체계적으로 통합-저장된 데이터의 모음이다. 이를 관리하는 소프트웨어가 DBMS다. 파일로 데이터를 흩어 관리할 때 생기는 중복-불일치 문제를 해결한다.

가장 널리 쓰이는 모델은 데이터를 표(테이블/릴레이션)로 표현한다.

  • 행(row/튜플) = 한 건의 레코드, 열(column/속성) = 항목.
  • 기본키(primary key): 각 행을 유일하게 식별하는 열(예: 학번).
  • 외래키(foreign key): 다른 테이블의 기본키를 참조해 테이블을 연결.
10.2 SQL

관계형 DB를 다루는 표준 언어.

sql
SELECT 이름, 학점 FROM 학생 WHERE 학과 = '컴퓨터과학' ORDER BY 학점 DESC;
  • 데이터 조작: SELECT(조회), INSERT(삽입), UPDATE(수정), DELETE(삭제).
  • 데이터 정의: CREATE, ALTER, DROP.
10.3 정규화와 트랜잭션
  • 정규화(normalization): 중복을 줄이고 이상 현상(갱신-삽입-삭제 이상)을 막도록 테이블을 분해하는 설계 과정.
  • 트랜잭션(transaction): 더 이상 쪼갤 수 없는 작업 묶음(예: 계좌 이체 = 출금 + 입금). ACID(원자성-일관성-고립성-지속성)를 보장해야 한다. "원자성"은 전부 성공하거나 전부 취소되어야 한다는 뜻.
  • NoSQL: 대용량-비정형 데이터를 위해 표 구조를 벗어난 DB(문서형-키-값형 등). 빅데이터-웹 서비스에서 함께 쓰인다.
Q. 트랜잭션의 ACID에서 "원자성(Atomicity)"이 뜻하는 바는?
정답 보기

11. 정보보안 기초

보강 단원 시험 필수 범위 아님 - 상위과목(컴퓨터보안) 대비 보강.

CIA 3요소

정보보안의 목표는 CIA 3요소로 요약된다.

  • 기밀성(Confidentiality): 허가된 사람만 정보를 본다.
  • 무결성(Integrity): 정보가 위변조되지 않는다.
  • 가용성(Availability): 필요할 때 정보를 쓸 수 있다.
11.1 암호화
방식키특징예
대칭키암호화/복호화에 같은 키빠름, 키 공유가 문제AES
공개키(비대칭키)공개키/개인키 한 쌍키 공유 안전, 느림RSA
  • 공개키 암호: 공개키로 잠그면 짝이 되는 개인키로만 열 수 있다. 이를 응용해 본인 인증을 하는 것이 전자서명.
  • 해시 함수: 임의 데이터를 고정 길이 값으로 바꾸는 일방향 함수(SHA-256 등). 무결성 검증과 비밀번호 저장에 쓴다(원본 복원 불가).
11.2 위협과 대응
  • 흔한 위협: 악성코드(바이러스-웜-랜섬웨어), 피싱, DDoS(대량 트래픽으로 서비스 마비), SQL 인젝션.
  • 기본 대응: 방화벽, 백신, 최신 보안 패치, 강한 비밀번호와 2단계 인증(2FA), 최소 권한 원칙.
  • 정보보안의 CIA 3요소는?
    기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability).
  • 대칭키와 공개키 암호의 차이는?
    대칭키는 같은 키로 암복호화(빠르나 키 공유가 문제, 예 AES), 공개키는 공개키/개인키 한 쌍(키 공유 안전하나 느림, 예 RSA).

12. 인공지능 개요

보강 단원 시험 필수 범위 아님 - 상위과목(인공지능-머신러닝-딥러닝) 대비 보강.

AI ⊃ 머신러닝 ⊃ 딥러닝

인공지능(AI)은 사람의 학습-추론-인지 능력을 기계로 구현하려는 분야다. 포함 관계를 기억하면 좋다: 인공지능 ⊃ 머신러닝 ⊃ 딥러닝.

  • 머신러닝(기계학습): 규칙을 사람이 일일이 코딩하지 않고, 데이터에서 패턴을 스스로 학습하는 방법.
    • 지도학습: 정답(레이블)이 있는 데이터로 학습(분류-회귀). 예: 사진에 "고양이/개" 라벨.
    • 비지도학습: 정답 없이 구조를 찾음(군집화). 예: 비슷한 고객 묶기.
    • 강화학습: 보상을 최대화하도록 시행착오로 학습. 예: 게임-로봇 제어.
  • 딥러닝: 인간 뇌의 신경망을 모방한 인공신경망을 깊게 쌓은 방법. 이미지 인식, 음성 인식, 자연어 처리에서 큰 성과를 냈다.
  • 생성형 AI / 대규모 언어 모델(LLM): 방대한 텍스트로 학습해 사람처럼 글-이미지를 생성하는 모델(ChatGPT 등). 환각(hallucination), 편향, 저작권 같은 한계와 윤리 쟁점도 함께 안고 있다.

여기서는 큰 그림만

방통대에서는 상위 학년의 인공지능-머신러닝-딥러닝 과목으로 이어지므로, 여기서는 용어와 큰 그림만 잡아 두면 된다.

Q. 지도학습-비지도학습-강화학습을 한 단어 키워드로 구분하면?
정답 보기
지도학습 = 정답(레이블) 있음(분류-회귀), 비지도학습 = 정답 없이 구조 찾기(군집화), 강화학습 = 보상 최대화(시행착오).

13. 소프트웨어공학과 컴퓨팅 윤리

보강 단원 시험 필수 범위 아님 - 상위과목(소프트웨어공학) 대비 보강.

13.1 소프트웨어공학

규모가 큰 소프트웨어를 체계적-경제적으로 개발-유지하기 위한 공학적 방법론이다.

  • 개발 생명주기(SDLC): 요구분석 -> 설계 -> 구현 -> 테스트 -> 유지보수.
  • 폭포수 모델: 단계를 순서대로 진행. 단순하지만 변경에 약함.
  • 애자일(Agile): 짧은 주기로 반복-점진 개발하며 변화에 유연하게 대응(스크럼 등).
  • 버전 관리(Git): 코드 변경 이력을 추적하고 협업하는 필수 도구. (이 레포도 Git으로 관리된다.)
13.2 컴퓨팅 윤리와 사회적 영향

기술이 사회에 미치는 영향이 커지면서 윤리적 책임도 커졌다.

  • 개인정보 보호: 데이터 수집-활용의 동의와 최소 수집 원칙(국내 개인정보보호법).
  • 지식재산권: 저작권, 오픈소스 라이선스(GPL, MIT 등) 준수.
  • 디지털 격차(digital divide): 정보 접근 기회의 불평등.
  • AI 윤리: 알고리즘 편향, 자동화로 인한 일자리 변화, 책임 소재, 딥페이크 등.
  • 정보윤리의 기본: 정확성, 타인 권리 존중, 보안 책임, 표절 금지.
Q. 폭포수 모델과 애자일의 가장 큰 차이는?
정답 보기
폭포수는 단계를 한 번에 순서대로 진행해 변경에 약하고, 애자일은 짧은 주기로 반복-점진 개발해 변화에 유연하게 대응한다(스크럼 등).
