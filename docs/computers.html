1. 정보사회와 컴퓨터

정보혁명, 데이터 vs 정보, 컴퓨터의 5대 기능

정보사회와 정보혁명

정보사회는 정보의 생산·처리·유통이 경제와 생활의 중심이 된 사회로, 농업혁명 → 산업혁명 → 정보혁명의 흐름이 낳은 결과다. 과거 핵심 자원이 토지/자본/노동이었다면, 정보사회에서는 "정보와 지식"이 가장 중요한 자원이 된다.

데이터와 정보의 차이 (핵심 구분)

구분데이터(Data)정보(Information)
의미가공 전의 원시 사실/값데이터를 가공해 의미를 부여한 결과
예시"23, 25, 21" (기온 숫자)"이번 주 평균 기온 23도, 어제보다 따뜻함"
관계입력처리 결과(출력)

가공 수준이 올라가면 데이터 → 정보 → 지식(knowledge) → 지혜(wisdom) 로 이어지는 DIKW 위계를 함께 기억하면 좋다.

컴퓨터 시스템의 구성과 기능

  • 컴퓨터의 5대 기능: 입력 - 기억(저장) - 연산 - 제어 - 출력.
  • 하드웨어 구분: 중앙처리장치(CPU), 기억장치(주기억/보조기억), 입출력장치.
  • 기본 동작 사이클: 입력 → 처리(연산/제어) → 출력. 필요한 데이터는 기억장치에 저장.

컴퓨터의 종류

  • 규모/용도: 슈퍼컴퓨터, 메인프레임, 미니컴퓨터(중형), 마이크로컴퓨터(PC), 스마트폰/임베디드 기기.
  • 데이터 표현 방식: 디지털 컴퓨터(대부분), 아날로그 컴퓨터, 하이브리드 컴퓨터.

언제 어디서나 컴퓨터와 네트워크를 쓸 수 있는 유비쿼터스 환경도 이 시대의 흐름이다(자세한 내용은 14장).

  • 정보사회의 핵심 자원은?
    정보와 지식 (과거: 토지/자본/노동)
  • 컴퓨터의 5대 기능
    입력 - 기억 - 연산 - 제어 - 출력
  • DIKW 위계
    데이터 → 정보 → 지식 → 지혜
Q. "오늘 측정한 기온 23, 25, 21"은 데이터인가 정보인가?

데이터. 가공 전의 원시 값이다. 이를 "평균 23도, 어제보다 따뜻함"처럼 의미를 부여하면 정보가 된다.

2. 컴퓨터와 통신산업의 발전

계산도구의 역사, 프로그램 내장방식, 세대 구분

계산도구의 역사

  • 초창기: 주판 → 파스칼의 기계식 계산기(덧셈/뺄셈) → 라이프니츠 계산기(곱셈/나눗셈) → 배비지의 해석기관(현대 컴퓨터의 개념적 원형).
  • 전자식: ENIAC(진공관 기반 최초의 대규모 전자식 컴퓨터). 단, ENIAC은 배선을 바꿔 프로그램했다.

프로그램 내장방식 (가장 중요한 개념)

폰 노이만 구조 / stored-program

프로그램(명령어)을 데이터처럼 기억장치에 저장해 두고, CPU가 순차적으로 꺼내 실행하는 방식. 오늘날 거의 모든 컴퓨터의 기본 구조다. 배선 교체 없이 소프트웨어만 바꿔 다른 작업을 수행할 수 있어 범용 컴퓨터가 탄생했다.

컴퓨터 산업의 발전 단계 (세대 구분)

세대핵심 소자특징
1세대진공관크고 발열 많음, ENIAC
2세대트랜지스터소형화, 신뢰성 향상
3세대집적회로(IC)더 작고 빨라짐
4세대고밀도 집적회로(LSI/VLSI), 마이크로프로세서PC 대중화
5세대인공지능/병렬처리 지향지능형 컴퓨터

컴퓨터와 통신의 결합

독립적으로 발전하던 컴퓨터 기술과 통신 기술이 만나 네트워크/인터넷 시대를 열었다. 산업의 미래는 AI, 클라우드, 모바일과의 융합 방향으로 나아간다.

  • 프로그램 내장방식을 제안한 구조 이름은?
    폰 노이만(von Neumann) 구조
  • 2세대 컴퓨터의 핵심 소자
    트랜지스터
Q. ENIAC의 프로그래밍 방식과 폰 노이만 구조의 결정적 차이는?
정답 보기
ENIAC은 배선을 직접 바꿔 프로그램했지만, 폰 노이만 구조는 프로그램을 데이터처럼 기억장치에 저장해 두고 꺼내 실행한다(소프트웨어 교체만으로 작업 변경 가능).

3. 처리장치와 데이터 처리

CPU 구성, 2진수 표현, 명령어 사이클, RAM/ROM

처리장치(CPU)의 구성

구성요소역할
제어장치(Control Unit)명령어를 해석하고 각 장치에 제어 신호 전달
연산장치(ALU)사칙연산과 논리연산(AND/OR/NOT, 비교 등) 수행
레지스터(Register)CPU 내부의 초고속 임시 저장 공간

데이터의 저장과 표현 (2진수)

  • 컴퓨터는 모든 데이터를 0과 1(2진수) 로 표현한다(전기 신호 on/off에 대응).
  • 비트(bit): 0 또는 1, 정보의 최소 단위. 바이트(byte): 8비트, 문자 1개 표현 단위.
  • 단위 위계: bit < byte < KB < MB < GB < TB.
  • 문자 표현: 아스키(ASCII, 영문 중심), 유니코드(Unicode, 한글 포함 전 세계 문자).
  • 음수/소수: 정수는 2의 보수, 실수는 부동소수점(floating point).

명령어 사이클

한 개의 명령어는 인출(Fetch) → 해석(Decode) → 실행(Execute) 과정을 거친다. 주기억장치는 주소(address)로 저장 위치를 구분하고, CPU는 그 주소로 데이터/명령어를 읽고 쓴다.

반도체기억장치

구분휘발성용도
RAM (주기억)휘발성(전원 꺼지면 소멸)실행 중인 프로그램/데이터 임시 저장
ROM비휘발성(전원 꺼져도 유지)부팅에 필요한 기본 프로그램 등 저장
  • 명령어 사이클 3단계
    인출(Fetch) → 해석(Decode) → 실행(Execute)
  • 1바이트는 몇 비트?
    8비트 (문자 1개 표현 단위)
  • 한글까지 표현하는 문자 코드
    유니코드(Unicode)
Q. 전원을 끄면 내용이 사라지는 휘발성 기억장치는 RAM과 ROM 중 무엇인가?
정답 보기
RAM. 휘발성이라 실행 중인 데이터를 임시 저장한다. ROM은 비휘발성으로 전원이 꺼져도 내용이 유지된다.

4. 컴퓨터의 입출력

입력/출력 장치 분류, soft/hard copy, 미래 인터페이스

중간과제물 출제 이력 단원

입출력장치 분류와 데이터 흐름을 정확히 정리해 둘 것. 입력 → 편집 → 처리 → 출력의 단계를 구분하는 것이 핵심이다.

구분방향예시
입력장치사람 → 컴퓨터키보드, 마우스, 터치스크린, 스캐너, 마이크, 카메라, 바코드/QR 리더, 센서
출력장치컴퓨터 → 사람모니터(디스플레이), 프린터, 스피커, 프로젝터

입력 데이터 편집과 출력 정보

  • 입력된 원시 데이터는 오류 검사, 형식 변환 등의 편집 과정을 거쳐 처리된다.
  • 출력은 화면 출력(soft copy)과 인쇄 출력(hard copy)으로 나뉜다.

입출력의 미래

음성 인식, 제스처 인식, VR/AR 인터페이스, 뇌-컴퓨터 인터페이스(BCI) 등 인간 친화적 입출력으로 발전 중이다.

  • soft copy vs hard copy
    soft copy = 화면 출력 / hard copy = 인쇄 출력
  • 스캐너는 입력장치? 출력장치?
    입력장치 (사람 → 컴퓨터)
Q. 다음 중 출력장치만 모은 것은? (모니터, 마이크, 프린터, 스캐너, 스피커)

5. 보조기억장치와 데이터 저장

주기억 vs 보조기억, 저장장치 진화, 버퍼/캐시, 클라우드

주기억장치 vs 보조기억장치

구분주기억장치(RAM)보조기억장치(HDD/SSD 등)
속도빠름느림(상대적)
용량작음큼
휘발성휘발성(전원 끄면 소멸)비휘발성(영구 보관)
용도실행 중 임시 저장장기 보관

저장장치의 종류와 진화

종류특징
자기테이프순차 접근(앞에서부터 차례로), 백업용
자기디스크(HDD)회전 자기 원판, 임의 접근(random access)

버퍼와 캐시

  • 버퍼(buffer): 속도 차이가 나는 두 장치 사이에서 데이터를 임시로 모아 두는 공간.
  • 캐시(cache): 자주 쓰는 데이터를 빠른 곳에 미리 두어 접근 속도를 높이는 기법.

클라우드 기반 저장

데이터를 내 기기가 아닌 인터넷상의 서버에 저장·공유(구글 드라이브, OneDrive, iCloud). 어디서나 접근 가능하지만 보안/네트워크 의존이라는 양면성이 있다.

  • 순차 접근만 가능한 저장장치는?
    자기테이프 (백업용)
  • 버퍼 vs 캐시의 목적 차이
    버퍼 = 속도 차 완충 / 캐시 = 자주 쓰는 데이터 미리 두어 속도 향상
Q. SSD가 HDD보다 충격에 강한 이유는?
정답 보기
SSD는 플래시 메모리(반도체) 기반이라 기계적으로 회전하는 부품이 없기 때문이다. HDD는 회전 자기 원판을 쓴다.

구분설명예시

소프트웨어 생명주기

유형설명

7. 데이터베이스

DB 개념, DBMS/SQL, 3단계 스키마, 데이터 웨어하우스

데이터베이스의 개념

데이터베이스(DB)는 여러 사용자가 공유할 수 있도록 통합/저장해 둔 데이터의 집합으로, 중복을 줄이고 일관성을 유지하는 것이 목표다. 파일 시스템 방식의 문제(데이터 중복, 불일치, 의존성)를 해결하기 위해 등장했다.

데이터베이스 관리 시스템(DBMS)

DBMS는 데이터베이스를 만들고 관리하며, 저장/검색/수정/삭제를 돕는 소프트웨어다(Oracle, MySQL, MS SQL Server). 사용자는 질의어(SQL)로 명령을 내린다.

3단계 스키마 구조

단계의미
외부 스키마사용자/응용 프로그램이 보는 부분적 관점(뷰)
개념 스키마전체 데이터베이스의 논리적 구조(조직 전체 관점)
내부 스키마실제 저장 방식(물리적 구조)

이 구조 덕분에 데이터 독립성(저장 방식이 바뀌어도 응용 프로그램 영향 최소화)이 확보된다.

장단점과 데이터 웨어하우스

  • 장점: 중복 최소화, 데이터 공유, 일관성/무결성 유지, 보안.
  • 단점: 시스템 구축/운영 비용, 복잡성, 장애 시 영향 큼.
  • 데이터 웨어하우스: 의사결정 지원을 위해 여러 곳의 데이터를 모아 분석용으로 저장하는 대규모 저장소(빅데이터 분석의 기반).
  • 데이터베이스에 명령을 내리는 질의어
    SQL
  • 3단계 스키마 (위에서 아래로)
    외부 스키마 → 개념 스키마 → 내부 스키마
Q. 실제 물리적 저장 방식을 정의하는 스키마는 무엇인가?
정답 보기
내부 스키마. 외부 스키마는 사용자 관점(뷰), 개념 스키마는 조직 전체의 논리적 구조다.

9. 4차 산업혁명과 미래사회

산업혁명 1~4차, AI/IoT/빅데이터/클라우드

중간과제물 단골 단원

사물인터넷(IoT) 적용 사례를 분야별로 4개 이상 구체적으로 적을 수 있어야 한다.

산업혁명의 단계

구분핵심 동력
1차증기기관(기계화)
2차전기(대량생산)
3차컴퓨터/인터넷(정보화, 자동화)
4차AI, IoT, 빅데이터, 초연결/초지능(융합)

4차 산업혁명 핵심 기술

  • 인공지능(AI): 학습/추론/판단을 하는 시스템. 머신러닝, 딥러닝 포함.
  • 사물인터넷(IoT): 사물에 센서/통신 기능을 부여해 인터넷에 연결(스마트 냉장고, 웨어러블, 스마트 가로등, 커넥티드 카).
  • 빅데이터: 방대하고 다양한 데이터를 빠르게 수집/분석해 가치를 뽑아냄. 3V(Volume, Variety, Velocity).
  • 클라우드 컴퓨팅: 컴퓨팅 자원을 인터넷으로 빌려 씀.
  • 그 외: 로봇, 자율주행, 3D 프린팅, 블록체인, AR/VR.
  • 빅데이터의 3V
    Volume(양), Variety(다양성), Velocity(속도)
  • 4차 산업혁명의 키워드
    초연결/초지능, 융합 (AI·IoT·빅데이터)

10. 정보통신

통신 기본 요소, 전송 방식, LAN/MAN/WAN, 프로토콜

정보통신의 기본 요소

송신자 - 전송매체 - 수신자, 그리고 약속(프로토콜)이 필요하다. 데이터를 전기/광 신호로 바꿔 전송하고, 받는 쪽에서 다시 복원한다.

정보통신 방식

분류 기준종류
전송 방향단방향(simplex), 반이중(half-duplex), 전이중(full-duplex)
신호 형태아날로그 전송, 디지털 전송
전송 단위직렬 전송, 병렬 전송

정보통신망(네트워크)의 유형

유형범위예
LAN근거리(건물/사무실)회사 내부망
MAN도시 규모도시 통신망
WAN광역(국가/대륙)인터넷 백본

토폴로지(망 구성 형태): 성형(star), 버스형(bus), 링형(ring), 망형(mesh).

통신 프로토콜

프로토콜(protocol)은 통신을 위한 약속(규칙)으로, 대표적으로 인터넷의 TCP/IP가 있다. 계층 모델(OSI 7계층 / TCP/IP 4계층)로 역할을 나눈다.

  • 양쪽이 동시에 송수신하는 전송 방향
    전이중(full-duplex) - 예: 전화 통화
  • 국가/대륙 규모의 광역 통신망
    WAN
Q. 무전기(한쪽이 말할 때 다른 쪽은 듣기만)는 어떤 전송 방향인가?
정답 보기
반이중(half-duplex). 양방향이지만 동시에는 불가능하다.

11. 인터넷

네트워크의 네트워크, IP/도메인/DNS/URL, 주요 서비스

인터넷의 개요

수많은 네트워크가 TCP/IP 프로토콜로 연결된 "네트워크의 네트워크". 특정 주인이 없는 개방형 구조다.

인터넷 주소

종류설명예
IP 주소인터넷에 연결된 각 기기의 숫자 주소203.0.113.1 (IPv4), IPv6는 더 큰 주소공간
도메인 이름사람이 외우기 쉬운 문자 주소www.knou.ac.kr
DNS도메인 이름과 IP 주소 변환 시스템전화번호부 역할
URL인터넷 자원의 위치 주소https://www.knou.ac.kr/page

인터넷 서비스와 응용

  • WWW(웹): HTTP 프로토콜, 브라우저로 하이퍼텍스트 문서 열람.
  • 이메일(SMTP/POP3/IMAP), FTP(파일 전송), 클라우드/스트리밍/SNS.
  • 응용: 전자상거래, 검색엔진, 소셜미디어, IoT, 웹 기반 서비스 전반.
  • 도메인 이름을 IP 주소로 바꿔 주는 시스템
    DNS (전화번호부 역할)
  • 웹에서 쓰는 문서 전송 프로토콜
    HTTP
Q. www.knou.ac.kr 는 IP 주소인가 도메인 이름인가?
정답 보기
도메인 이름. 사람이 외우기 쉬운 문자 주소이며, DNS가 이를 실제 IP 주소로 변환한다.

12. 정보보안

CIA 3대 목표, 악성코드, 사회공학, 보안 강화

정보보안의 3대 목표 (CIA)

목표의미
기밀성(Confidentiality)인가된 사람만 정보에 접근
무결성(Integrity)정보가 임의로 변조되지 않음
가용성(Availability)필요할 때 정보/서비스를 사용 가능

위협 요소

  • 해킹: 시스템에 무단 침입해 정보를 빼내거나 파괴.
  • 악성 프로그램(malware): 바이러스(다른 파일에 기생), 웜(스스로 복제/전파), 트로이목마(정상 프로그램 위장), 랜섬웨어(파일 암호화 후 금전 요구), 스파이웨어.
  • 사회공학적 공격: 피싱(가짜 사이트/메일), 파밍, 스미싱(문자 이용).

보안 강화 방법

  • 암호화(encryption): 데이터를 알아볼 수 없게 변환. 대칭키/공개키 방식.
  • 인증: 비밀번호, 생체인증, OTP, 2단계 인증(2FA).
  • 방화벽, 백신, 보안 패치 최신화, 백업.
  • 정보보안 3대 목표 (CIA)
    기밀성 · 무결성 · 가용성
  • 파일을 암호화한 뒤 금전을 요구하는 악성코드
    랜섬웨어
  • 문자메시지를 이용한 사기 공격
    스미싱
Q. 스스로 복제해 네트워크로 퍼지는 악성코드는? (바이러스 / 웜 / 트로이목마)
정답 보기
웜(worm). 바이러스는 다른 파일에 기생하고, 트로이목마는 정상 프로그램으로 위장한다.

13. 멀티미디어

멀티미디어 특징, 이미지/사운드/동영상 표현, 압축, A/D 변환

멀티미디어란

텍스트, 이미지, 사운드, 애니메이션, 동영상 등 두 가지 이상의 미디어를 디지털로 통합해 다루는 것. 특징은 디지털화, 통합성, 상호작용성(interactivity).

디지털 미디어의 표현

미디어핵심 개념대표 형식
이미지비트맵(픽셀 기반) vs 벡터(수식 기반), 해상도, 색 깊이JPEG, PNG, GIF, BMP
사운드샘플링(아날로그→디지털), 샘플링 비율MP3, WAV
동영상프레임의 연속, 압축(코덱)MP4, AVI
  • 압축: 데이터 크기를 줄이는 기술. 무손실(원본 완전 복원) vs 손실(일부 정보 버려 더 작게).
  • 아날로그 → 디지털 변환: 표본화(sampling) → 양자화(quantization) → 부호화(encoding).

멀티미디어의 활용

교육(이러닝), 게임, 영화/방송, 가상현실(VR)/증강현실(AR), 디지털 광고 등.

  • 확대해도 깨지지 않는 이미지 방식
    벡터(수식 기반). 비트맵은 픽셀 기반이라 확대 시 계단현상
  • A/D 변환 3단계
    표본화(sampling) → 양자화(quantization) → 부호화(encoding)
Q. 원본을 완전히 복원할 수 있는 압축 방식은 무손실인가 손실인가?
정답 보기
무손실 압축. 손실 압축은 일부 정보를 버려 더 작게 만들지만 원본을 완전히 복원하지 못한다.

14. 유비쿼터스

유비쿼터스의 개념

유비쿼터스(ubiquitous)는 라틴어로 "어디에나 존재한다"는 뜻이다. 사용자가 의식하지 않아도 일상 곳곳에 컴퓨터가 스며들어 언제 어디서나 정보를 활용하는 환경을 말한다. 특성은 보이지 않음(invisible), 어디에나 있음, 자연스러운 상호작용, 네트워크 연결.

관련 기술과 사물인터넷(IoT)

  • RFID/NFC(근거리 무선 인식), 센서, 무선 네트워크가 기반.
  • 사물인터넷(IoT): 사물들이 서로 연결되어 데이터를 주고받음. 유비쿼터스 비전을 실현하는 핵심.

스마트시티와 전망

IoT, 빅데이터, AI를 도시 인프라(교통, 에너지, 안전, 환경)에 적용해 효율적이고 지속가능하게 운영하는 도시(스마트 교통 신호, 스마트 미터, 스마트 주차). 헬스케어, 스마트홈, 웨어러블, 자율주행 등으로 확장 중이다.

  • "어디에나 존재한다"는 뜻의 컴퓨팅 환경
    유비쿼터스(ubiquitous)
  • 유비쿼터스 비전을 실현하는 핵심 기술
    사물인터넷(IoT)

Q. 안드로이드와 iOS의 가장 큰 개방성 차이는?

안드로이드는 오픈소스 기반으로 여러 제조사가 채택하는 개방형이고, iOS는 애플 자사 기기 전용의 폐쇄적/통합적 OS다.

자주 나오는 개념 구분

데이터 vs 정보 / 주기억 vs 보조기억(휘발성) / 시스템 SW vs 응용 SW / LAN vs WAN / 비트맵 vs 벡터 / 안드로이드 vs iOS.

분야별 사례를 구체적으로 적을 수 있어야 한다. 특히 IoT 사례, 4차 산업혁명 기술은 중간과제물 단골이다.

단답형 대비 포인트

2진수, 비트/바이트 단위, ASCII/유니코드, 샘플링/압축 개념은 단답형으로 나올 수 있다.

Q. (종합) 폰 노이만 구조의 핵심 아이디어를 한 문장으로 말해 보세요.
정답 보기
프로그램(명령어)을 데이터처럼 기억장치에 저장해 두고 CPU가 순차적으로 꺼내 실행하는 방식(stored-program). 배선 교체 없이 소프트웨어만 바꿔 다른 작업을 할 수 있어 범용 컴퓨터가 가능해졌다.
Q. (종합) 주기억장치와 보조기억장치를 휘발성·속도·용도로 비교하세요.
정답 보기
주기억(RAM)은 휘발성·빠름·작은 용량으로 실행 중 임시 저장에, 보조기억(HDD/SSD)은 비휘발성·상대적으로 느림·큰 용량으로 장기 보관에 쓰인다.
