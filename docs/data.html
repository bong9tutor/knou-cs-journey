1. 데이터 분석과 컴퓨터

척도 4단계, 모집단/표본, 분석 워크플로우

1장은 도구 실습에 앞서 "그 도구로 무엇을 다루는지"를 정리하는 단계다. 데이터 타입(척도)을 알아야 어떤 통계량/그래프를 쓸지 정하고, 모집단과 표본을 구분해야 "내 데이터로 어디까지 말할 수 있는가"를 판단한다. 1장은 이후 모든 장의 공통 문법이다.

핵심 질문 세 가지: (1) 계산해도 되는 숫자인가, 이름표일 뿐인 숫자인가? (2) 전체인가, 일부인가(모집단 vs 표본)? (3) 무엇을 거쳐 결론에 도달하는가(워크플로우)?

측정 척도 4가지 수준

척도특징예시의미 있는 요약
명목(nominal)순서 없는 분류성별, 혈액형, 지역코드최빈값
순서(ordinal)순서는 있으나 간격 불일정만족도(상/중/하), 학점, 선호 순위중앙값까지
등간(interval)간격 일정, 절대 0 없음섭씨 온도, 연도덧셈/뺄셈(배수는 무의미)
비율(ratio)절대 0 있음, 배수 비교 가능키, 몸무게, 소득, 나이모든 사칙연산

암기 팁

명목 < 순서 < 등간 < 비율 순으로 "정보량"이 커진다. 위 단계 연산은 아래 단계에서도 항상 가능하다(비율은 다 됨).

꼭 구분할 짝

  • 변수 vs 관측치: 변수는 측정 항목(키, 성별), 관측치는 한 개체의 측정값 묶음(한 사람 = 한 행).
  • 범주형 vs 연속형: 범주형은 빈도/비율로 요약, 연속형은 평균/표준편차로 요약.
  • 모수 vs 통계량: 모집단 특성치는 모수(parameter), 표본 특성치는 통계량(statistic).
  • 기술통계 vs 추론통계: 기술통계는 가진 데이터를 요약/묘사, 추론통계는 표본으로 모집단을 추정/검정.

분석 워크플로우 7단계

문제 정의 → 데이터 수집 → 정제(전처리) → 탐색(EDA) → 분석/모형화 → 해석/시각화 → 보고

엑셀로 손 기술통계(실습 B)

연속형 열에는 =AVERAGE/=MEDIAN/=STDEV.S/=COUNT를, 명목형 열에는 =COUNTA로 개수만 센다. 범주형 = 막대그래프, 연속형 분포 = 히스토그램이라는 짝을 기억한다.

척도 분류 퀴즈

Q. 다음을 명목/순서/등간/비율로 분류하라 - 학번, 시험 점수(0~100), 학년(1~4), 우편번호, 섭씨 기온.
정답 보기
학번 = 명목(식별용 이름표), 시험 점수 = 비율(절대 0과 배수 의미), 학년 = 순서(간격 불일정), 우편번호 = 명목(연산 무의미), 섭씨 기온 = 등간(절대 0 없음, 20도가 10도의 두 배 아님).

2. 데이터의 입력과 점검

결측치/이상치 처리와 Tidy Data

"쓰레기를 넣으면 쓰레기가 나온다(Garbage In, Garbage Out)." 분석가 시간의 60~80%는 모델링이 아니라 데이터를 읽어 들이고 결측치/이상치를 점검하고 정제(cleaning)하는 데 들어간다(정확한 수치라기보다 정제 비중이 그만큼 크다는 통념). 2장은 분석 전 단계의 기초 체력이다.

핵심 개념

개념설명
정형 데이터 구조행 = 관측 단위 1건, 열 = 변수 1개, 셀 = 하나의 값. 모든 분석 도구의 전제.
결측치 표기엑셀 빈 칸, R의 NA, pandas의 NaN/None. 빈칸/0/"모름" 문자열을 반드시 구분.
결측치 처리행/열 삭제(dropna, na.omit), 대치(수치형 평균/중앙값, 범주형 최빈값, 모델 예측).
이상치(outlier)분포에서 비정상적으로 먼 값. 탐지: 상자그림, IQR 규칙, Z-점수.
IQR 규칙Q1 - 1.5×IQR 미만 또는 Q3 + 1.5×IQR 초과를 이상치 후보로 본다.

Tidy Data 3원칙 (Hadley Wickham)

(1) 각 변수는 하나의 열, (2) 각 관측치는 하나의 행, (3) 각 값은 하나의 셀. "넓은 형식(wide)"을 "긴 형식(long)"으로 바꾸는 것이 핵심 변환이다.

실습 C: pandas로 점검/정제

python
import pandas as pd
df = pd.read_csv("data.csv", encoding="utf-8")  # 깨지면 "cp949"

# 첫 점검 3종 세트
df.shape; df.head(); df.info(); df.describe()

# 결측치 처리
df.isnull().sum()                              # 열별 결측 개수
df2 = df.dropna()                              # 결측 행 삭제
df["소득"] = df["소득"].fillna(df["소득"].median())  # 중앙값 대치

# 중복 제거 + IQR 이상치 필터
df = df.drop_duplicates()
q1, q3 = df["소득"].quantile([0.25, 0.75])
iqr = q3 - q1
mask = (df["소득"] >= q1 - 1.5*iqr) & (df["소득"] <= q3 + 1.5*iqr)
df_clean = df[mask]

4. 문서작성

한글/워드 표/수식/보고서 구조 (서론-분석-결론)

분석은 "남이 읽고 판단할 수 있는 문서"로 전달될 때 가치가 생긴다. 표가 어그러지거나 수식이 깨져 보이거나 흐름이 뒤죽박죽이면 독자는 결론을 신뢰하지 않는다. 4장은 정확한 표, 올바른 수식, 논리적 보고서 구조를 다룬다. (교재 화면은 한글 2018 기준이며 최신 버전은 메뉴 위치/명칭이 조금 다를 수 있다.)

핵심 기능

기능요점
표 편집셀 병합/분할, 제목 행 음영, 숫자 오른쪽 정렬+소수점 통일. 표 안 계산식 =SUM(A1:A5).
캡션 관행표 제목은 표 위, 그림 제목은 그림 아래. 일련번호(<표 1>, [그림 1])와 출처 표기.
시계열 차트연도가 가로축인 시계열은 분산형(X,Y) "직선 및 표식" 차트. 꺾은선과 달리 X축 간격이 실제 수치 간격으로 반영.
수식 편집기한글: 입력 → 수식, 단축키 Ctrl+N, M. 분수/첨자/루트/시그마/적분/행렬.

분석 보고서 구조

서론(주제/배경/목적/데이터 출처) → 분석(데이터 설명 → 방법 → 표/그래프 → 해석) → 결론(핵심 발견 + 시사점 + 한계/향후 과제). 표/그림에는 일련번호와 캡션, 출처를 반드시 단다.

실습 C: 수식 편집기로 평균/표준편차

text
평균:    bar x = {1 over n} sum_{i=1}^{n} x_i
표준편차: s = sqrt{ {1 over {n-1}} sum (x_i - bar x)^2 }

표 안 합계는 빈 셀에 커서를 두고 =SUM(B2:B4)처럼 범위를 지정한다. 원본 숫자를 바꾸면 다시 계산해 값이 갱신되는지 확인한다.

5. 엑셀 데이터 처리

참조, 조회/조건 집계 함수, 정렬/필터/피벗

데이터 분석의 출발점은 거창한 언어가 아니라 엑셀이다. 절대/상대 참조, 조건부 집계, 피벗테이블은 이후 R의 dplyr이나 Python pandas의 groupby로 그대로 이어진다.

셀 참조

형태의미
A1상대 참조 - 복사하면 참조가 함께 이동
$A$1절대 참조 - 행/열 모두 고정
$A1, A$1혼합 참조 - 행 또는 열만 고정

F4 순환

F4를 반복하면 A1 → $A$1 → A$1 → $A1 순으로 전환된다. VLOOKUP 범위는 F4로 절대 참조 고정이 필수(고정 안 하면 범위가 밀려 #N/A).

주요 함수

함수역할
VLOOKUP(찾을값, 범위, 열번호, FALSE)첫 열에서 찾아 같은 행 다른 열 값. 실무는 거의 항상 FALSE(정확 일치).
IF(조건, 참, 거짓)조건 분기
COUNTIF / SUMIF / AVERAGEIF조건 만족 개수 / 합계 / 평균 (다중조건은 ...IFS)
CORREL(범위1, 범위2)두 변수의 상관계수(-1 ~ +1)

표본 vs 모집단 외우기

STDEV = STDEV.S = 표본(n-1), STDEVP = STDEV.P = 모집단(n). 이름 끝 P(Population)가 모집단 신호다. 분산도 동일: VAR/VAR.S = 표본, VARP/VAR.P = 모집단.

실습 A: 조건 함수

text
C2: =IF(B2>=60, "합격", "불합격")     점수 분기
    =COUNTIF(B2:B11, ">=60")          합격자 수
    =SUMIF(B2:B11, ">=60")            합격자 점수 합
    =AVERAGE(B2:B11)  =MEDIAN(B2:B11)  =STDEV.S(B2:B11)

피벗테이블은 [삽입 → 피벗 테이블]로 행 영역에 등급, 값 영역에 점수를 넣고 "값 필드 설정"에서 합계를 평균으로 바꾼다. 같은 결과를 AVERAGEIF로도 구해 일치를 확인한다.

6. 엑셀 그래프 그리기

차트 선택 기준, 산점도/추세선, 단순선형회귀

그래프는 숫자 덩어리를 시각 신호(길이/위치/기울기/흩어짐)로 바꿔 "무엇이 크고 어떻게 변하고 관계가 있는가"를 한눈에 보이게 한다. 데이터 성격에 맞는 차트를 고르는 판단력이 핵심이다(잘못된 차트는 데이터를 왜곡한다).

차트 종류 = 데이터 성격

차트언제 쓰나
원형(파이)전체에 대한 구성비. 합이 100%, 항목 5개 이내일 때.
막대(세로/가로)여러 항목 간 값 크기 비교. 이름이 길면 가로막대.
꺾은선시간 흐름에 따른 추세(시계열의 기본).
히스토그램한 수치형 변수의 구간별 분포. 막대 사이가 붙어 있다.
산점도(분산형)두 수치형 변수 사이의 관계.

차트 왜곡 주의

세로축 시작값을 0이 아닌 곳에서 끊으면 작은 차이가 과장된다. 상관관계는 인과관계가 아니다(상관이 있어도 원인-결과로 단정 금지).

단순선형회귀 (추세선의 정체)

산점도의 "선형 추세선"이 곧 단순선형회귀의 결과다. 회귀식은 y = a + bx: 기울기 b는 x가 1 늘 때 y의 평균 변화량(방향/세기), 절편 a는 직선이 세로축과 만나는 위치. R제곱(결정계수)은 회귀직선이 y의 분산 중 몇 %를 설명하는지를 0~1로 나타내며, 단순선형회귀에서는 상관계수 r의 제곱과 같다.

text
=SLOPE(y범위, x범위)      기울기 b
=INTERCEPT(y범위, x범위)  절편 a
=RSQ(y범위, x범위)        R제곱

엑셀 추세선에서 [추세선 추가 → 선형] 후 "수식을 차트에 표시"와 "R-제곱 값을 차트에 표시"를 체크하면 차트에 회귀식과 R²가 함께 나온다.

R로 한 줄 회귀

r
fit <- lm(y ~ x, data = df)  # y를 x로 설명하는 단순선형회귀
summary(fit)                 # (Intercept)=a, x=기울기 b, Multiple R-squared=R제곱
Q. 다음 네 상황에 가장 적절한 차트는? (1) 연도별 출생아 수 변화, (2) 17개 시도 인구 비교, (3) 한 반 30명의 키 분포, (4) 가구 소비지출 항목별 구성비.
정답 보기
(1) 꺾은선 - 시간 추세, (2) 막대 - 항목 비교, (3) 히스토그램 - 분포, (4) 원형 - 구성비(합 100%).

7. 엑셀 활용

날짜/문자열 함수, 재무함수(원리합계), 해 찾기

현실 데이터는 날짜가 20240315나 텍스트로 들어오고, 이름/주소가 한 칸에 섞여 있으며, "월 적립 30만원을 3년 부으면 만기에 얼마인가" 같은 의사결정 질문이 따른다. 7장은 엑셀을 전처리 + 의사결정 + 시각화 도구로 쓴다.

날짜/문자열 함수

함수역할
YEAR/MONTH/DAY(날짜)연/월/일 추출 (시계열 집계의 기초)
DATEDIF(시작, 끝, "Y"/"M"/"D")두 날짜 사이 햇수/개월/일수 (나이/근속)
EOMONTH(시작일, 개월)해당 월의 말일
LEFT/MID/RIGHT + FIND구분자 기준으로 길이가 제각각인 셀 자르기

날짜의 정체

엑셀에서 날짜는 1900-01-01을 1로 세는 일련번호(숫자)다. 그래서 날짜 - 날짜 = 일수가 그냥 뺄셈으로 된다.

재무함수(원리합계, 복리)

text
=FV(A1/12, A2*12, -A3)   적금 만기금액(미래가치)
=PV(이율, 기간, 정기납입)  현재가치
=PMT(C1/12, C2*12, C3)   대출 월상환금

부호와 단위 규칙

내가 내는 돈은 음수(-), 받는 돈은 양수(+). 월 단위면 이율은 연이율/12, 기간은 년수*12.

해 찾기 / 목표값 찾기

  • 목표값 찾기(Goal Seek): 변수 1개를 바꿔 수식 결과를 특정 값으로 (예: 목표 만기를 맞추는 월 납입액). [데이터 → 가상 분석 → 목표값 찾기].
  • 해 찾기(Solver): 변수 여러 개 + 제약조건 하에서 목표를 최대/최소/특정값으로 만드는 최적화. 추가 기능이라 별도 로드 필요. 구성: 목표 셀 / 변수 셀 / 제한 조건.
