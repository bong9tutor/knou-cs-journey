1. 컴퓨터와 프로그래밍의 이해

데이터/정보/프로그램, 하드웨어와 소프트웨어, 인터프리터 vs 컴파일러

데이터, 정보, 프로그램

  • 데이터(data): 가공되지 않은 원시 사실. 예) 숫자 23, 문자 'A'.
  • 정보(information): 데이터를 처리해 의미를 부여한 것. 예) "오늘 기온 23도".
  • 프로그램(program): 컴퓨터가 수행할 명령들의 모음. 프로그래밍은 그 명령을 작성하는 일.

컴퓨터는 입력(input) -> 처리(process) -> 출력(output)의 흐름으로 동작합니다. 프로그래밍 언어는 사람이 이해하는 표현을 컴퓨터가 실행할 수 있는 형태로 옮기는 도구입니다.

하드웨어와 소프트웨어

  • 하드웨어: CPU(연산), 메모리(RAM, 임시 저장), 보조기억장치(SSD/HDD, 영구 저장), 입출력장치.
  • 소프트웨어: 시스템 소프트웨어(운영체제 등)와 응용 소프트웨어(우리가 만드는 프로그램).

인터프리터 vs 컴파일러

출석수업대체과제물의 단골 서술형 주제입니다. 둘 다 고급 언어 소스를 컴퓨터가 실행할 형태로 옮기는 번역기라는 점은 같습니다.

구분인터프리터컴파일러
대표 언어Python, JavaScriptC, C++, Java(바이트코드)
변환 방식소스를 한 줄씩 해석하며 즉시 실행소스 전체를 기계어로 번역한 뒤 실행
실행 파일별도 산출물 없음(원본 + 인터프리터 필요)실행 파일(예: .exe) 생성
실행 속도상대적으로 느림(매번 해석)상대적으로 빠름(미리 번역)
오류 발견 시점실행 중 그 줄에 도달해야 발견컴파일 단계에서 한꺼번에 발견
작성-실행 사이클짧음(바로 실행, 배우기 쉬움)길음(컴파일 단계 필요)
이식성인터프리터만 있으면 어디서나 실행플랫폼별로 다시 컴파일해야 함

핵심 차이: 인터프리터는 실행 시점에 한 줄씩 번역하고, 컴파일러는 실행 전에 전체를 한 번에 번역합니다. 이 차이에서 속도, 오류 발견 시점, 이식성이 갈립니다. 파이썬은 인터프리터 언어이므로 작성-실행 사이클이 짧고 배우기 쉽지만 실행 속도는 상대적으로 느립니다.

python
print("Hello, KNOU!")
예상 출력 보기

print()는 화면에 값을 출력하는 내장 함수입니다. 이 한 줄이 가장 작은 완성된 파이썬 프로그램입니다.

2. 파이썬 환경과 첫 프로그램

Colab/로컬, REPL과 스크립트, 주석/출력/입력

실행 환경: Colab과 로컬

KNOU 강의는 설치 없이 브라우저에서 바로 쓰는 Google Colab을 권장합니다. 셀 단위로 코드를 실행하며 결과를 즉시 확인할 수 있습니다. 로컬에서 학습하려면 python.org에서 파이썬을 설치하고 IDLE 또는 VS Code를 씁니다.

3. 변수, 자료형, 연산자

동적 타입 변수, 기본 자료형, 형변환, 산술/비교/논리/복합 연산자

변수와 할당

변수는 값을 담는 이름입니다. 파이썬은 동적 타입이라 선언 시 자료형을 명시하지 않고, 할당된 값에 따라 자료형이 정해집니다.

python
x = 10          # 정수
pi = 3.14       # 실수
name = "파이썬"  # 문자열
is_ok = True    # 불리언
a = b = c = 0   # 다중 할당
p, q = 1, 2     # 언패킹 할당
p, q = q, p     # 두 값 교환 (임시변수 불필요)
print(p, q)     # 2 1

변수 이름 규칙: 영문/숫자/밑줄, 숫자로 시작 불가, 예약어(if, for 등) 사용 불가. 관례상 소문자와 밑줄을 씁니다(student_count).

형변환(type casting)

python
print(int("100") + 1)   # 101  문자열 -> 정수

print(str(2026) + "년")  # 2026년  정수 -> 문자열

int()는 내림이 아니라 0 방향 절단

int()는 소수부를 잘라내 0 방향으로 절단(truncate)합니다. 항상 작은 쪽(음의 무한대 방향)으로 내리는 math.floor()와 다릅니다. 양수는 같지만 음수에서 갈립니다.

python
import math
print(int(3.9), math.floor(3.9))     # 3 3    양수는 동일
print(int(-3.9), math.floor(-3.9))   # -3 -4  음수에서 차이!

bool()에서 0, 0.0, "", [], {}, None은 거짓으로 평가됩니다.

문자열 기본과 슬라이싱

문자열은 문자들의 순서가 있는 불변(immutable) 시퀀스입니다. 슬라이싱은 s[start:stop:step] 형식으로 start는 포함, stop은 제외합니다.

python
s = "Python"
print(len(s))     # 6  길이
print(s[0])       # P  인덱싱(0부터)
print(s[-1])      # n  음수 인덱스(뒤에서부터)
print(s + "3")    # Python3  연결
print(s * 2)      # PythonPython  반복
print("y" in s)   # True  포함 여부

t = "ABCDEFG"
print(t[1:4])    # BCD
print(t[:3])     # ABC  처음부터
print(t[4:])     # EFG  끝까지
print(t[::2])    # ACEG  2칸씩
print(t[::-1])   # GFEDCBA  뒤집기

문자열 메서드

python
t = "  Hello, World  "
print(t.strip())           # "Hello, World"  양끝 공백 제거
print(t.strip().upper())   # "HELLO, WORLD"
print("a,b,c".split(","))   # ['a', 'b', 'c']  분리 -> 리스트
print("-".join(["2026", "06", "08"]))  # 2026-06-08  결합
print("banana".replace("a", "o"))  # bonono
print("banana".count("a"))  # 3
print("banana".find("n"))   # 2  첫 위치(없으면 -1)
print("Python".startswith("Py"))  # True
print("file.txt".endswith(".txt")) # True

문자열은 불변이므로 이런 메서드는 원본을 바꾸지 않고 새 문자열을 반환합니다.

문자열 포매팅(f-string)

가장 권장되는 방식은 f-string(파이썬 3.6+)입니다.

python
name, score = "홍길동", 92.5
print(f"{name}님 점수: {score}점")  # 홍길동님 점수: 92.5점
print(f"{score:.1f}")               # 92.5  소수 첫째 자리
print(f"{1234567:,}")               # 1,234,567  천 단위 콤마
print(f"{255:b}, {255:x}")          # 11111111, ff  2진수/16진수
print(f"{0.875:.1%}")               # 87.5%  백분율
print(f"{name:>10}|")               # "       홍길동|"  오른쪽 정렬

리스트(list)

python

특수 메서드와 연산자 오버로딩 __add__ 심화/참고

                
__str__은 print()나 str()이 쓰는 사람이 읽기 좋은 표현을 정의합니다. __add__처럼 + 같은 연산자에 대한 동작을 클래스가 직접 정의하는 것을 연산자 오버로딩이라 합니다. 시험에서는 __init__과 __str__, 일반 메서드 정의가 핵심입니다.

                

                    
python

                    
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __str__(self):
        return f"({self.x}, {self.y})"

    def __add__(self, other):           # + 연산자 재정의(오버로딩)
        return Point(self.x + other.x, self.y + other.y)

p = Point(1, 2)
q = Point(3, 4)
print(p)         # (1, 2)
print(p + q)     # (4, 6)

                


                
상속(inheritance)과 다형성

                
기존 클래스(부모)의 기능을 물려받아 확장합니다. 같은 info() 호출이 객체에 따라 다르게 동작하는 것을 다형성(polymorphism)이라 합니다.

                

                    
python

                    
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return "..."

    def info(self):
        return f"{self.name}: {self.speak()}"

class Cat(Animal):              # Animal을 상속
    def speak(self):           # 메서드 오버라이딩(재정의)
        return "야옹"

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # 부모 생성자 호출
        self.breed = breed

    def speak(self):
        return "멍멍"

c = Cat("나비")
d = Dog("바둑이", "진돗개")
print(c.info())   # 나비: 야옹
print(d.info())   # 바둑이: 멍멍
print(d.breed)    # 진돗개

class Account:

import random

def number_game():
    answer = random.randint(1, 100)
    tries = 0
    print("1~100 사이 숫자를 맞혀 보세요!")
    while True:
        try:
            guess = int(input("추측: "))
        except ValueError:
            print("정수를 입력하세요.")
            continue
        tries += 1
        if guess < answer:
            print("더 큰 수입니다.")
        elif guess > answer:
            print("더 작은 수입니다.")
        else:
            print(f"정답! {tries}번 만에 맞혔습니다.")

def make_report(scores):
    """scores: {이름: 점수} 딕셔너리를 받아 요약 문자열을 반환."""
    if not scores:
        return "데이터가 없습니다."
    values = list(scores.values())
    avg = sum(values) / len(values)
    top = max(scores, key=scores.get)
    bottom = min(scores, key=scores.get)
    lines = [f"{name}: {score}" for name, score in scores.items()]
    lines.append(f"평균: {avg:.1f}")
    lines.append(f"최고: {top}({scores[top]})")
    lines.append(f"최저: {bottom}({scores[bottom]})")
    return "\n".join(lines)

scores = {"홍길동": 90, "김철수": 75, "이영희": 88}
print(make_report(scores))

핵심 요소: 클래스/객체, 클래스 변수, 딕셔너리로 규칙 표현, 상태 관리.

import random

class RockPaperScissors:
    HANDS = ["가위", "바위", "보"]
    WINS = {"가위": "보", "바위": "가위", "보": "바위"}  # 키가 값을 이김

    def __init__(self):
        self.user_win = 0
        self.comp_win = 0

    def play(self, user_hand):
        comp_hand = random.choice(self.HANDS)
        if user_hand == comp_hand:
            result = "무승부"
        elif self.WINS[user_hand] == comp_hand:
            result = "승리"
            self.user_win += 1
        else:
            result = "패배"
            self.comp_win += 1
        return f"나:{user_hand} vs 컴퓨터:{comp_hand} -> {result}"

game = RockPaperScissors()
print(game.play("바위"))
print(f"전적: {game.user_win}승 {game.comp_win}패")
